## 유형 A: 신기능 개발

### 예제 1 — LinkDots 초기 구현 (대형 기능)

```markdown
# 계획서: LinkDots (Flow Free 클론)

## 1. 개요
동일한 색상의 점들을 그리드 위에서 연결하는 미니멀한 퍼즐 게임입니다. 모든 점의 쌍을 연결해야 하며, 완벽한 승리를 위해서는 그리드의 모든 칸이 선(파이프)으로 채워져야 합니다.

## 2. 핵심 기능
- [ ] **그리드 및 보드 시스템**:
    - 가변 그리드 크기 (초기 5x5).
    - 좌표 기반 셀 관리.
- [ ] **점(Dot) 배치**:
    - 고정된 위치에 동일한 색상의 점 쌍 배치 (레벨 데이터).
- [ ] **선 그리기 (상호작용)**:
    - 터치 또는 마우스 드래그를 통해 점 사이의 연속적인 선 생성.
    - 선의 색상은 시작점의 색상과 일치해야 함.
- [ ] **로직 및 규칙**:
    - [ ] **교차 금지**: 선은 서로 겹칠 수 없음. 기존 선을 가로지르면 기존 선이 끊어짐.
    - [ ] **동일 색상 연결**: 동일한 색상의 점끼리만 연결 가능.
    - [ ] **전체 점유**: 승리를 위해 그리드의 모든 칸이 선으로 채워져야 함.

## 3. 기술 스택
- **프레임워크**: Phaser 3 (루트의 phaser.js 활용).
- **언어**: JavaScript (ES6+).
- **에셋**: 절차적 그래픽 (원형 점, 사각형/직선 파이프).

## 4. 상세 작업 목록
- [ ] 5x5 그리드의 Phaser 게임 인스턴스 초기화.
- [ ] 데이터 관리를 위한 `Grid` 및 `Cell` 클래스 생성.
- [ ] 경로 그리기 로직 구현 (입력 처리, 라인 렌더링).
- [ ] 경로 교차 시 기존 선 끊기(Pipe Breaking) 로직 구현.
- [ ] 승리 조건 판정 로직 (모든 점 연결 + 모든 칸 점유).
- [ ] 초기화 버튼 및 이동 횟수 카운터 추가.
- [ ] 3~5개의 샘플 레벨 데이터 구축.

## 5. 성공 지표
- 사용자가 버그 없이 모든 점의 쌍을 연결할 수 있음.
- 완벽한 승리 시 그리드가 100% 채워짐을 감지함.
- 경로 그리기가 매끄럽고 그리드 좌표를 정확히 따름.
- 경로 중복 시 기존 경로가 정상적으로 제거됨.

### 예제 2 — 레벨 시스템 확장 (데이터/시스템 기능)

```markdown
# 계획서: 레벨 시스템 확장 (LinkDots)

## 1. 개요
현재 `MainScene` 내부에 하드코딩된 레벨 데이터를 외부 JSON 파일로 분리하고, 다수의 레벨을 순차적으로 플레이하거나 선택할 수 있는 시스템을 구축합니다.

## 2. 핵심 기능
- [ ] **외부 데이터 분리**: 레벨 데이터를 `assets/levels/levels.json` 파일로 이동.
- [ ] **레벨 로더 구현**: Phaser의 `load.json`을 사용하여 데이터를 동적으로 로드.
- [ ] **레벨 관리자(LevelManager) 추가**:
    - 현재 레벨 추적.
    - 다음 레벨로 전환 로직.
    - 레벨 팩(5x5, 6x6 등) 구분 지원.
- [ ] **진행 시스템**:
    - 레벨 클리어 시 'Next Level' 버튼 표시.
    - 마지막 레벨 클리어 시 축하 메시지 및 처음으로 가기.

## 3. 기술 스택
- **데이터 형식**: JSON
- **에셋 관리**: Phaser Loader (Scene.load)
- **로직 처리**: Singleton 패턴 또는 Scene 프로퍼티를 활용한 레벨 관리

## 4. 상세 작업 목록
- [ ] 레벨 데이터를 구조화하여 `assets/levels/levels.json` 생성.
- [ ] `MainScene`의 `preload()` 단계에서 JSON 파일 로드 로직 추가.
- [ ] `MainScene`의 `create()` 시점에 JSON 데이터를 파싱하여 그리드 초기화.
- [ ] 승리 팝업에 'NEXT LEVEL' 버튼 추가 및 클릭 이벤트 구현.
- [ ] 레벨 전환 시 기존 그리드와 경로를 완벽히 정리(Cleanup)하는 로직 강화.

## 5. 성공 지표
- 외부 JSON 파일 수정만으로 게임 레벨을 추가/변경할 수 있음.
- 'Next Level' 클릭 시 다음 레벨이 오류 없이 로드되고 플레이 가능함.
- 모든 레벨을 순차적으로 플레이하는 흐름이 매끄러움.

## 유형 B: 버그 수정

### 예제 1 — 파이프 삭제 시 UI 동기화 버그

```markdown
# 계획서: 파이프 삭제 시 배경색 및 점유율 동기화 버그 수정

## 1. 개요
파이프의 시작점을 터치하여 기존 경로를 초기화했을 때, 셀 배경색과 파이프 점유율(%) 데이터가 즉시 갱신되지 않고 잔상이 남는 문제를 해결합니다.

## 2. 버그 원인 분석
- **UI 갱신 누락**: `MainScene.js`의 `pointerdown` 이벤트 핸들러에서 `pathManager.startPath()` 호출 후 `gridManager.draw()`와 `updateHUD()`를 호출하지 않아, 드래그를 시작하기 전까지 이전 배경색이 유지됨.
- **길이 1 경로의 잔존**: 사용자가 점을 터치만 하고 떼었을 때, `PathManager`에는 길이가 1인 경로가 저장되어 있어 해당 셀이 계속 `isOccupied = true` 상태로 남음.

## 3. 해결 전략
- **이벤트 핸들러 보완**: `MainScene`의 `pointerdown` 및 `pointerup` 시점에도 그리드 재그리기와 HUD 업데이트를 명시적으로 수행합니다.
- **경로 자동 정리**: `PathManager.endPath()` 메서드에서 경로의 길이가 2 미만인 경우, 해당 색상의 경로를 완전히 삭제(`clearPath`)하도록 로직을 강화합니다.

## 4. 상세 작업 목록
- [ ] `src/scenes/MainScene.js`: `pointerdown` 이벤트 핸들러에 `gridManager.draw()` 및 `updateHUD()` 호출 추가.
- [ ] `src/classes/PathManager.js`: `endPath()` 메서드에 길이가 1인 경로를 감지하여 삭제하는 로직 추가.
- [ ] `src/scenes/MainScene.js`: `pointerup` 이벤트 핸들러에 `gridManager.draw()` 호출 추가.
- [ ] 수정 후 점 터치 시 즉시 배경색이 사라지고 점유율이 줄어드는지 확인.

## 5. 성공 지표
- 점을 터치하는 순간 기존 파이프와 배경색이 즉시 사라짐.
- 파이프가 사라진 만큼 파이프 % 수치가 실시간으로 감소함.
- 연결되지 않은 점(Dot) 주위에는 배경색이 남지 않음.

### 예제 2 — 점유율 계산 초기값 오류 (코드 인용 포함)

```markdown
# 계획서: 파이프 점유율 계산 버그 수정

## 1. 개요
게임 시작 시 또는 파이프가 없는 상태에서 파이프 점유율(Pipe %)이 0%가 되지 않는 현상을 수정합니다.

## 2. 버그 원인 분석
- **`GridManager.js`의 `getFillPercentage()`**:
  ```javascript
  if (this.cells[y][x].isOccupied || this.cells[y][x].dotColor) {
      occupied++;
  }
  ```
  `dotColor`가 있는 칸(시작/종료 점)을 항상 점유된 것으로 처리하고 있습니다.
  5x5 그리드에서 5쌍(10개)의 점이 있다면 시작부터 40%가 표시됩니다.

## 3. 해결 전략
- **점유 기준 변경**: `dotColor` 유무와 관계없이, 실제 파이프가 그려진 상태를 나타내는 `isOccupied` 값만 기준으로 계산하도록 수정합니다.

## 4. 상세 작업 목록
- [ ] `src/classes/GridManager.js` 내 `getFillPercentage()` 메서드: `|| this.cells[y][x].dotColor` 조건 제거.
- [ ] `isAllFilled()` 메서드 검토: 승리 조건도 `isOccupied` 중심인지 확인.
- [ ] 수정 후 게임 초기 상태에서 점유율이 0%인지 검증.

## 5. 성공 지표
- 레벨 시작 시 파이프 점유율이 정확히 **0%**로 표시됨.
- 파이프를 그릴 때마다 점유율이 실시간으로 증가함.
- 모든 칸을 파이프로 채웠을 때만 **100%**가 됨.

- **MainScene 수정**: 새로운 UI 레이아웃 적용 및 하단 버튼 이벤트 바인딩.

# 계획서: 경로 곡선 라운딩 처리

## 2. 버그 원인 분석

- **마디(Joints)**: 모든 셀 좌표 위치에 `fillCircle`을 그려 자연스럽게 둥글게 처리합니다.
