# 게임 분석 스킬 - 레퍼런스 게임 분석

## 목적

**웹 검색으로 레퍼런스 게임을 직접 조사**하여 실행 가능한 디자인 인사이트와 재사용 가능한 시스템 패턴으로 변환합니다.

사전 지식에만 의존하지 말고 반드시 `WebSearch`와 `WebFetch`를 사용하여 실제 데이터를 수집한 후 분석합니다.

**추출 대상**:
- 핵심 게임플레이 루프 (마이크로~메가 계층)
- MDA 분석 (메카닉 → 다이나믹스 → 에스테틱스)
- 플레이어 동기 구조
- 시스템 아키텍처
- 보상 타이밍과 빈도
- 난이도 곡선 및 온보딩
- 경제 및 수익화 구조

---

## 분석 이론 기반

### MDA 프레임워크

Robin Hunicke, Marc LeBlanc, Robert Zubek의 공식 게임 분석 프레임워크(GDC 2001~2004).

| 레이어 | 정의 | 분석 방향 |
|--------|------|-----------|
| **Mechanics (메카닉)** | 규칙, 알고리즘, 플레이어가 취할 수 있는 행동 단위 | 디자이너가 직접 설계 |
| **Dynamics (다이나믹스)** | 메카닉과 플레이어 입력이 상호작용하며 창발하는 행동 | 런타임에서 발현 |
| **Aesthetics (에스테틱스)** | 플레이어가 경험하는 감정적 반응 | 플레이어가 먼저 경험 |

> **핵심**: 디자이너는 메카닉 → 다이나믹스 → 에스테틱스 순으로 설계하지만,
> 플레이어는 에스테틱스 → 다이나믹스 → 메카닉 순으로 경험한다.
> 분석 시 **플레이어 관점(역방향)**과 **디자이너 관점(순방향)** 모두를 적용한다.

**8가지 에스테틱 유형 (LeBlanc)**:
Sensation(감각), Fantasy(환상), Narrative(서사), Challenge(도전), Fellowship(유대감), Discovery(발견), Expression(표현), Submission(몰입)

### 스킬 아톰 (Skill Atom)

플레이어의 학습과 숙련을 설명하는 피드백 루프 단위.

```
액션 (Action) → 시뮬레이션 (Simulation) → 피드백 (Feedback) → 모델링 (Modeling) → 액션
```

- 플레이어는 새로운 스킬 습득에서 무의식적 만족을 느낀다
- 게임은 이 스킬 아톰들이 연결된 방향성 그래프로 구성된다
- 분석 시 "어떤 스킬 아톰이 코어인가"를 먼저 식별한다

## 분석 워크플로우

사용자가 레퍼런스 게임을 제공하면 **반드시 다음 순서**로 분석합니다.

### 0단계: 웹 검색으로 게임 정보 수집 (필수)

**기본 정보 수집**:
```
WebSearch: "[게임명] game mechanics"
WebSearch: "[게임명] game analysis"
WebSearch: "[게임명] review [현재연도]"
```

**심층 정보 수집**:
```
WebSearch: "[게임명] core loop breakdown"
WebSearch: "[게임명] progression system explained"
WebSearch: "[게임명] wiki"
WebSearch: "[게임명] monetization model"
WebSearch: "site:reddit.com [게임명] game design"
WebSearch: "site:gamedeveloper.com [게임명]"
WebSearch: "site:deconstructoroffun.com [게임명]"
```

**WebFetch로 상세 내용 읽기**:
- 검색 결과 중 분석 아티클, 위키, 팬 사이트를 WebFetch로 직접 읽기
- 수치(레벨 수, 보상 주기, 리텐션 지표 등) 적극 수집

> **중요**: 검색 결과가 불충분하면 추가 검색어를 시도합니다. 수집된 정보 출처(URL)를 분석 보고서에 명시합니다.

### 1~6단계: 수집된 정보를 기반으로 분석

1. **에스테틱 목표 식별**: 게임이 플레이어에게 주려는 감정 경험 (MDA 역방향)
2. **핵심 루프 추출**: 마이크로~메가 계층 루프 식별
3. **MDA 분해**: 에스테틱을 만들어내는 다이나믹스, 그것을 구성하는 메카닉
4. **진행 구조 분석**: 플레이어 성장과 난이도 곡선
5. **의사결정 구조**: 플레이어가 내리는 의미 있는 선택
6. **수익화 연결점**: 경제 구조와 플레이어 동기의 교차점

## 출력 구조

항상 다음 구조로 분석 결과를 제공합니다.

### 1. 게임 개요

```
게임: [게임 제목]
장르: [장르]
플랫폼: [플랫폼]
개발사: [개발사]
출시일: [출시일]
타겟 유저: [타겟 유저층]

핵심 에스테틱 (LeBlanc 8유형 중): [해당 유형들]
게임 경험 요약: [한 문장으로 핵심 경험 설명]

조사 출처:
- [URL 1] — [출처 설명]
- [URL 2] — [출처 설명]
```

### 2. MDA 분석

에스테틱(감정 목표)에서 역방향으로 분해합니다.

**에스테틱 (플레이어가 경험하는 감정)**:
- [이 게임이 유발하는 핵심 감정 경험 2~3가지]

**다이나믹스 (에스테틱을 만드는 창발 행동)**:
- [에스테틱 A를 만드는 런타임 패턴]
- [에스테틱 B를 만드는 런타임 패턴]

**메카닉 (다이나믹스를 구성하는 규칙)**:
- [다이나믹스를 만드는 구체적인 규칙/수치/행동 단위]

> 이 역방향 분해를 통해 "왜 재미있는가"를 구조적으로 설명합니다.

### 3. 핵심 게임플레이 루프

가장 작은 반복 단위부터 큰 단위까지 계층적으로 설명.

**마이크로 루프** (가장 짧은 반복 단위, 수초~30초):
```
[핵심 인터랙션 한 사이클]
예) 이동 → 적 조우 → 공격 → 결과 확인 → 반복        (액션)
예) 퍼즐 상태 파악 → 이동/조작 → 결과 확인 → 반복    (퍼즐)
예) 오브젝트 발견 → 2개 병합 → 상위 오브젝트 생성     (머지)
예) 장애물 인식 → 반응 입력 → 생존/실패 확인          (하이브리드 캐주얼)
```

**코어 루프** (5~15분, 한 세션의 핵심 순환):
```
[단기 목표 달성 사이클 — 한 전투/레벨/라운드]
예) 레벨 클리어 → 별/보상 획득 → 다음 레벨 선택
예) 머지 보드 진행 → 미션 목표 달성 → 저택 복원 진행
```

**매크로 루프** (세션 단위 이상, 메타 진행):
```
[장기 목표 달성 사이클 — 스토리 진행, 영구 성장, 컬렉션 완성]
예) 월드 클리어 → 신규 월드 언락 → 더 어려운 퍼즐 도전
예) 런 완료/실패 → 메타 업그레이드 → 재도전
예) 건물 업그레이드 → 레벨업 → 서사 챕터 해금
```

**메가 루프** (수일~수주, 리텐션 구조):
```
[장기 리텐션을 담당하는 시즌/이벤트/소셜 사이클]
예) 주간 이벤트 → 한정 보상 수집 → 다음 시즌 대기
예) 길드 보스 레이드 → 사회적 기여 → 소속감 강화
```

**분석 포인트**:
- 플레이어 동기: 왜 루프를 반복하는가?
- 보상 타이밍: 언제, 얼마나 자주 만족감을 주는가?
- 코어-메타 연결 강도: 두 루프가 얼마나 긴밀하게 결합되어 있는가?

### 4. 플레이어 동기 분석

**Octalysis 8가지 핵심 동인** 중 이 게임이 활용하는 요인을 식별.

| 동인 | 활용 여부 | 게임 내 표현 방식 |
|------|-----------|------------------|
| Epic Meaning (서사적 의미) | | |
| Development (성장/성취) | | |
| Creativity & Feedback (창의성) | | |
| Ownership (소유욕) | | |
| Social Influence (사회적 영향) | | |
| Scarcity (희소성/조급함) | | |
| Unpredictability (호기심/예측불가) | | |
| Loss Avoidance (손실 회피) | | |

**플레이어 여정 단계별 주요 동인**:
- Discovery (첫 접촉): [어떤 동인으로 플레이어를 끌어들이는가]
- Onboarding (초반 10분): [어떤 동인으로 첫 경험을 강화하는가]
- Scaffolding (중반): [어떤 동인으로 지속 플레이를 유도하는가]
- Endgame (장기): [어떤 동인으로 고인물을 붙잡는가]

### 5. 플레이어 의사결정 구조

플레이어가 내리는 **의미 있는 선택**을 식별.

**주요 의사결정 유형**:
- **자원 배분**: 한정된 자원을 어떻게 나눌 것인가
- **경로/전략 선택**: 가능한 옵션 중 무엇을 선택할 것인가
- **리스크/보상 판단**: 위험을 감수할 것인가, 안전하게 갈 것인가
- **타이밍 판단**: 언제 특정 행동을 취할 것인가

**좋은 의사결정의 특징**:
- 명확한 트레이드오프
- 즉각적인 피드백 (스킬 아톰 완결)
- 전략적 깊이

### 6. 게임플레이 시스템 분해

게임의 핵심 시스템을 독립 모듈로 분리하여 설명합니다.

각 시스템에 대해 다음을 기술합니다:
- **역할**: 이 시스템이 게임에서 하는 기능
- **입력**: 플레이어 액션 또는 다른 시스템에서 받는 데이터
- **출력**: 플레이어에게 돌아가는 결과 또는 다른 시스템에 전달하는 데이터
- **주요 파라미터**: 밸런싱에 영향을 주는 핵심 수치/규칙

> 분석한 게임에서 실제로 존재하는 시스템으로 구성합니다.
> - 퍼즐 게임: 퍼즐 생성·검증·힌트 시스템, 레벨 목표 시스템
> - 하이브리드 캐주얼: 코어 루프 시스템과 메타 레이어를 별도 시스템으로 분리
> - 소셜 요소가 있는 경우: 소셜 루프 시스템 (공격/방어/협력)

### 7. 보상 구조

**보상 타이밍과 비율이 내용만큼 중요합니다.** 무엇을 주느냐보다 언제, 얼마나 자주 주느냐가 행동 강화에 더 큰 영향을 미칩니다.

**즉각 보상** (마이크로 루프 완결 시):
- [행동 직후 돌아오는 즉각적 피드백 — 사운드, 이펙트, 수치 변화]

**단기 보상** (코어 루프 완결 시):
- [레벨/라운드 클리어 등 단기 목표 달성 보상]

**장기 보상** (매크로/메가 루프):
- [지속적인 플레이를 유도하는 누적·영구 보상]

**변수 보상 설계**:
- 예측 가능 보상(Fixed Reward) vs 무작위 보상(Variable Reward) 비율
- Variable Reward는 도파민 반응을 극대화하지만 과도하면 신뢰 하락
- 퍼즐 게임의 Near Miss 설계: 실패 직전 상황이 "거의 됐다"는 감각을 주도록 설계되는가

**보상 빈도 분석**:
- 보상 주기를 정량적으로 기술 (예: 레벨업 1회당 평균 소요 시간)
- 보상 간격이 플레이어 동기에 미치는 영향 분석

### 8. 난이도 곡선

**난이도 증가 방식**:
- [시간, 스테이지, 레벨 등 진행에 따라 난이도가 어떻게 변하는가]
- 퍼즐 게임: 규칙 복잡도 증가, 제한된 이동 수/시간, 새로운 장애물 유형 도입
- 하이브리드 캐주얼: 장애물 밀도/속도 상승, 패턴 다양화

**플레이어 성장 속도**:
- [초반/중반/후반 각 구간에서 플레이어 역량과 난이도의 관계]
- 학습 곡선이 급격한지, 완만한지 기술

**주요 난이도 분기점**:
- [플레이어가 실패 또는 전략 전환을 경험하는 구간]
- 퍼즐 장르 참고 기준: 첫 100레벨은 튜토리얼+쉬운 레벨, 이후 레벨당 평균 3회 이상 시도

**이탈 위험 구간**:
- 이탈이 집중될 수 있는 구간과 그 이유
- 해당 구간의 리텐션 완충 장치 (힌트, 세이브 포인트, 부스터 등)

### 9. UX / 피드백 디자인

**핵심 액션 피드백**:
- 주요 행동에 대한 시청각(+촉각) 피드백 방식
- 스킬 아톰의 피드백 단계가 충분히 명확한가

**정보 전달**:
- HUD 구성 요소 및 역할
- 중요 정보를 어떤 방식으로 전달하는가 (상시 표시 vs 맥락적 표시)

**온보딩/튜토리얼 구조**:
- 신규 플레이어가 핵심 메카닉을 배우는 방식
- 퍼즐: 규칙을 직접 가르치는가 vs 발견하게 두는가 (Monument Valley식 암묵적 튜토리얼 vs Candy Crush식 명시적 안내)
- 하이브리드 캐주얼: 첫 세션(5분) 내 핵심 훅 노출 시점

**인지 부하 관리**:
- 한 번에 처리해야 할 정보량이 적절히 분산되어 있는가
- 새로운 메카닉 도입 시 기존 메카닉과의 충돌을 어떻게 방지하는가

### 10. 콘텐츠 구조

**콘텐츠 단위**:
- [스테이지, 레벨, 챕터, 월드 등 콘텐츠의 기본 단위]
- 단위당 평균 플레이 시간과 재시도 횟수

**콘텐츠 다양성 확보 방식**:
- 절차적 생성 / 수동 제작 / 혼합 방식
- 재플레이 가치를 높이는 구조적 장치

**콘텐츠 언락 구조**:
- 플레이어가 새 콘텐츠에 접근하는 방식 (선형 vs 분기 vs 개방형)
- 드리블링(Drip-feeding) 전략: 다음 콘텐츠를 예고하여 기대감을 유지하는가

### 11. 경제 및 수익화 구조

**자원 종류와 역할**:
- [소프트 화폐, 하드 화폐, 에너지, 소모 아이템 등 목록과 역할]
- 다중 통화 구조라면 각 통화의 획득 난이도와 소비 심리 분리

**자원 획득 경로**:
- 플레이 보상 / 시간 경과 재생 / 소셜 보상 / 유료 구매

**자원 소비처**:
- 자원을 어디에 사용하는가, 각 소비가 주는 만족감

**수익화 접점**:
- 광고 노출 시점과 방식 (보상형 광고: 자발적, 인터스티셜: 강제)
- 인앱 결제 항목과 플레이어 동기(진행 가속/독점 콘텐츠/소유욕)의 연결
- 코어 루프와 수익화 접점의 연결 강도: 수익화가 코어 경험을 방해하는가, 보완하는가

**에너지 시스템** (해당하는 경우):
- 에너지 재생 주기와 최대 보유량
- 세션 길이 조절 효과 분석

**경제 밸런싱**:
- 자원 희소성, 인플레이션 방지 장치, 플레이어 진행 속도

### 12. 디자인 기둥

게임을 지탱하는 3~5개의 핵심 원칙:

- **[원칙 1]**: 이 게임이 추구하는 가장 중요한 경험
- **[원칙 2]**: 플레이어를 지속적으로 끌어들이는 핵심 동기
- **[원칙 3]**: 게임이 차별화되는 독특한 특성
- (추가 원칙이 있다면 기술)

### 13. 재사용 가능한 디자인 패턴

분석한 게임에서 추출한, 다른 프로젝트에도 적용할 수 있는 패턴들.

각 패턴에 대해 다음을 기술합니다:
- **패턴 이름**: 패턴을 간결하게 표현하는 이름
- **설명**: 이 패턴이 무엇인가
- **효과**: 플레이어 경험(에스테틱)에 어떤 영향을 주는가
- **적용 예시**: 이 패턴이 잘 드러나는 게임 내 사례

**참고 — 장르별 대표 패턴**:
- 퍼즐: Near Miss 설계, 규칙 복잡도 점진 도입, 암묵적 튜토리얼
- 하이브리드 캐주얼: 코어-메타 단일 화폐 연결 (Royal Match의 별), 코어+메타 이중 세그먼트 타겟팅
- 공통: 변수 보상 루프, 메타 진행 보상, 세션 길이 조절 에너지 시스템

### 14. 프로토타입 범위 추천

**MVP**:
- 핵심 에스테틱(감정 목표)을 검증하기 위한 최소 기능 목록
- 코어 스킬 아톰 1~2개만으로 재미의 핵을 확인할 수 있는가
- 이 단계에서 반드시 확인해야 할 게임플레이 가설

**알파**:
- MVP에서 확장된 콘텐츠와 시스템
- 코어 루프 완결 + 메타 루프 초안이 연결되는 단계
- 첫 플레이테스트 가능 수준

**베타**:
- 콘텐츠 완성도, 밸런싱, UX 다듬기
- 수익화 구조 및 에너지 시스템 포함
- 외부 플레이테스트 및 리텐션 측정 가능 수준

### 15. 추가 분석 (다중 레퍼런스)

여러 레퍼런스 게임 제공 시:

**비교 분석**:
- 공통 메카닉/다이나믹스 식별
- 에스테틱 목표의 차이점 분석
- 각 게임의 코어-메타 연결 방식 비교

**하이브리드 디자인 제안**:
- 게임 A의 코어 루프 + 게임 B의 메타 레이어 조합
- 새로운 조합의 가능성과 리스크 (에스테틱 충돌 여부)

[위의 구조를 따라 전체 분석 제공, 출처 URL 포함]

## 모범 사례

1. **웹 검색 필수**: 분석 전 반드시 WebSearch/WebFetch로 정보 수집 — 사전 지식만으로 분석하지 않기
2. **MDA 역방향 시작**: 플레이어가 느끼는 감정(에스테틱)부터 식별한 뒤 메카닉으로 역추적
3. **출처 명시**: 분석 보고서에 참조한 URL을 반드시 포함
4. **복수 출처 교차 검증**: 리뷰, 개발자 블로그, 커뮤니티 의견을 통합하여 편향 방지
5. **보상 타이밍 정량화**: 무엇을 주는지보다 언제, 얼마나 자주 주는지를 수치로 기술
6. **코어-메타 연결 강도 평가**: 두 루프의 결합이 느슨하면 장기 리텐션에 취약
7. **불확실성 표시**: 공개 정보로 추정한 수치는 "추정값"임을 명시
8. **플레이어 관점 우선**: 왜 재미있는지, 왜 중독성 있는지 경험 중심으로 분석
