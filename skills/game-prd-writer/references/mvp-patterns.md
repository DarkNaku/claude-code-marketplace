# 장르별 MVP 패턴

장르별 MVP 최소 기능 명세, 수용 기준 작성 가이드, 자주 하는 실수 정리.

---

## 수용 기준(Acceptance Criteria) 작성 원칙

기능 명세의 수용 기준은 다음 기준을 충족해야 합니다:

- **검증 가능**: "잘 작동한다" (❌) → "3초 이내에 피드백이 표시된다" (✅)
- **주어 명확**: 플레이어 행동과 시스템 반응을 구분
- **엣지 케이스 포함**: 최솟값, 최댓값, 경계 조건 명시
- **부정 조건 포함**: 해서는 안 되는 동작도 기술

**수용 기준 작성 템플릿**:
```
Given [초기 상태]
When  [플레이어 행동]
Then  [시스템 반응]
```

예시:
```
Given 플레이어가 적에게 피격될 수 있는 상태일 때
When  적이 플레이어 충돌체와 접촉하면
Then  플레이어 HP가 지정된 수치만큼 감소하고 피격 이펙트가 즉시 재생된다
```

## 장르별 MVP 최소 기능 명세

### 로그라이크 서바이벌

**핵심 경험**: 자동 공격으로 적을 처치하며 레벨업 시 스킬을 선택하는 빌드 조합의 재미

**핵심 경험 가설**: "레벨업 선택 화면에서 플레이어가 '어떤 걸 골라야 하지?' 고민하면 핵심 재미가 있다"

**MVP 필수 기능**:

| 기능 | 플레이어 시나리오 | 핵심 수용 기준 |
|------|-----------------|--------------|
| 캐릭터 이동 | 플레이어는 입력 방향으로 캐릭터를 이동시킬 수 있다 | 입력 후 0.1초 이내 이동 시작, 8방향 이동 모두 동작 |
| 자동 공격 (1종) | 캐릭터는 일정 주기로 가장 가까운 적을 자동 공격한다 | 공격 범위 내 적이 없으면 공격 없음, 범위 내 있으면 반드시 공격 |
| 적 스폰 | 시간에 따라 적이 화면 외곽에서 등장한다 | 게임 시작 5초 이내 첫 적 등장, 이후 지속적으로 증가 |
| 적 추적 AI (1종) | 적은 항상 플레이어 방향으로 이동한다 | 플레이어 위치 변화 후 1초 이내 방향 전환 |
| 경험치 시스템 | 적을 처치하면 경험치가 떨어지고 수집하면 레벨업 수치가 오른다 | 경험치 아이템이 플레이어 일정 거리 내 자동 흡수됨 |
| 레벨업 선택 | 레벨업 시 게임이 멈추고 3가지 스킬 중 하나를 선택한다 | 선택 전까지 게임 진행 정지, 선택 즉시 재개 |
| HP / 사망 | 적 접촉 시 HP 감소, 0이 되면 게임 오버 | 피격 직후 무적 시간 존재, HP 0 시 즉시 게임 오버 화면 |
| 재시작 | 게임 오버 후 재시작할 수 있다 | 재시작 시 모든 상태 초기화 |

**의도적 제외 (MVP)**:
- 무기 다양성 → 단일 무기 1종으로 충분
- 메타 진행 (영구 업그레이드) → 알파
- 다양한 적 종류 → 알파
- 세이브/로드 → 알파
- 배경음악, 완성된 이펙트 → 베타

### 덱빌딩 카드게임

**핵심 경험**: 카드 조합으로 시너지를 만들어 강력한 한 방을 터뜨리는 전략적 만족감

**핵심 경험 가설**: "플레이어가 '이 카드랑 저 카드가 잘 맞네' 시너지를 스스로 발견하면 핵심 재미가 있다"

**MVP 필수 기능**:

| 기능 | 플레이어 시나리오 | 핵심 수용 기준 |
|------|-----------------|--------------|
| 에너지 시스템 | 플레이어는 턴마다 에너지를 가지며 카드 플레이 시 소비한다 | 에너지 부족 시 카드 플레이 불가, 시각적 표시 |
| 카드 드로우/플레이/버리기 | 플레이어는 손패에서 카드를 선택하여 플레이할 수 있다 | 드로우 덱 소진 시 버린 덱 재활용 |
| 공격 카드 | 카드를 플레이하면 적에게 명시된 수치만큼 데미지가 들어간다 | 데미지 수치가 즉시 시각/숫자로 표시됨 |
| 방어 카드 | 카드를 플레이하면 이번 턴 받는 피해를 막는 블럭이 쌓인다 | 블럭은 턴 종료 시 사라짐, 적 공격 시 블럭이 먼저 소모됨 |
| 적 인텐트 | 적이 다음 턴에 취할 행동이 미리 표시된다 | 플레이어 턴 시작 시 적의 다음 행동 아이콘 표시 |
| 시작 덱 | 게임 시작 시 고정된 덱을 가진다 | 공격 카드 5장 + 방어 카드 5장 (또는 동등한 구성) |
| 카드 보상 | 전투 승리 후 3장 중 1장을 덱에 추가할 수 있다 | 덱에 추가 후 다음 전투에서 드로우 가능 |
| 전투 승리/패배 | 적 HP를 0으로 만들거나 자신의 HP가 0이 되면 전투가 끝난다 | 패배 시 게임 오버 화면 출력 |
| 연속 전투 | 전투 3~5회 연속으로 진행할 수 있다 | 전투 사이 체력 회복 없음 (HP 유지) |

**의도적 제외 (MVP)**:
- 캐릭터 다양성 → 알파
- 유물/렐릭 시스템 → 알파
- 맵/경로 선택 → 알파
- 카드 업그레이드 → 알파
- 상점, 캠프파이어 → 알파

### 3매치 퍼즐

**핵심 경험**: 제한된 이동 수 안에 목표를 달성하는 긴장감과 연쇄 콤보의 카타르시스

**핵심 경험 가설**: "이동 수 2~3개 남은 상황에서 클리어하면 '아슬아슬하게 됐다'는 쾌감이 나온다"

**MVP 필수 기능**:

| 기능 | 플레이어 시나리오 | 핵심 수용 기준 |
|------|-----------------|--------------|
| 타일 스와이프 | 플레이어는 인접한 타일을 스와이프하여 자리를 바꿀 수 있다 | 같은 색 3개 이상 연결 시에만 이동 성립, 아니면 원위치 |
| 타일 제거 및 낙하 | 3개 이상 연결된 타일은 제거되고 위 타일이 떨어진다 | 제거 후 0.3초 이내 낙하 애니메이션 완료 |
| 연쇄 처리 | 낙하 후 새로 3개 이상 연결되면 자동으로 추가 제거된다 | 연쇄가 끝날 때까지 이동 수 소모 없음 |
| 이동 수 제한 | 플레이어는 지정된 이동 수 내에 목표를 달성해야 한다 | 이동 수 0이 되면 즉시 실패 처리 |
| 레벨 목표 | 레벨마다 특정 타일 N개를 제거하는 목표가 있다 | 목표 달성 시 즉시 클리어 판정 |
| 클리어/실패 판정 | 목표 달성 시 클리어, 이동 수 소진 시 실패한다 | 클리어/실패 화면이 즉시 표시됨 |
| 레벨 진행 | 클리어 후 다음 레벨로 진행할 수 있다 | 최소 5개 레벨 순서대로 진행 |
| 재시도 | 실패 후 같은 레벨을 다시 시작할 수 있다 | 재시도 시 보드 초기 상태로 리셋 |

**의도적 제외 (MVP)**:
- 파워업 종류 다양화 → 알파 (MVP는 1종 또는 없음으로 시작)
- 별 평가 시스템 → 알파
- 메타 레이어 (왕국 건설 등) → 알파
- 생명/에너지 시스템 → 알파

### 머지 (Merge) 게임

**핵심 경험**: 오브젝트를 병합하여 새로운 것을 발견하는 수집과 탐색의 재미

**핵심 경험 가설**: "처음 보는 오브젝트가 병합으로 등장할 때 '이게 나오는구나' 발견의 반응이 나오면 핵심 재미가 있다"

**MVP 필수 기능**:

| 기능 | 플레이어 시나리오 | 핵심 수용 기준 |
|------|-----------------|--------------|
| 보드 및 오브젝트 배치 | 플레이어는 고정된 보드 위에 오브젝트가 놓여 있는 것을 본다 | 보드 크기 내 오브젝트가 시각적으로 명확히 구분됨 |
| 드래그 앤 드롭 병합 | 같은 오브젝트 2개를 겹치면 상위 오브젝트 1개로 합쳐진다 | 병합 불가한 오브젝트 위에 드롭 시 원위치, 오류 피드백 |
| 병합 결과 피드백 | 병합 시 새 오브젝트 등장 애니메이션이 재생된다 | 새 오브젝트가 처음 등장하면 시각적 강조 효과 |
| 오브젝트 소환 | 플레이어는 에너지를 소비하여 새 오브젝트를 보드에 추가한다 | 보드가 가득 찬 경우 소환 불가 + 시각적 안내 |
| 에너지 시스템 | 에너지는 시간에 따라 자동 재충전된다 | 에너지 0 시 소환 불가, 잔여량과 재충전 타이머 표시 |
| 미션 목표 | 특정 오브젝트 N개를 제출하면 미션이 완료된다 | 목표 오브젝트가 항상 화면에 표시됨 |
| 미션 완료 보상 | 미션 완료 시 시각적 변화(배경/공간 변화)가 나타난다 | 변화가 즉시 화면에 반영됨 |

**의도적 제외 (MVP)**:
- 서사/스토리 → 알파 (MVP는 시각적 변화만)
- 다중 오브젝트 카테고리 → 알파
- 다중 통화(코인/젬) → 알파
- 이벤트 시스템 → 베타

### 하이브리드 캐주얼 (코어+메타)

**핵심 경험**: 짧은 코어 루프를 반복하며 눈에 보이는 메타 목표가 조금씩 달성되는 진행감

**핵심 경험 가설**: "코어 루프 1회 완료 후 메타 목표가 조금이라도 전진했다는 것이 시각적으로 보이면 계속 하고 싶어진다"

**MVP 접근법**:
- 코어 루프와 메타 루프를 **하나의 단일 화폐**로 연결
- MVP에서 코어 루프 + 메타 1단계 진행만 구현
- "코어 클리어 → 메타 변화"의 인과가 즉각적으로 느껴지는지 검증

**코어-메타 연결 체크**:
```
□ 코어 루프 보상이 메타 루프 진행에 직접 사용되는가?
□ 메타 변화가 코어 루프 완료 후 5초 이내에 시각적으로 표시되는가?
□ 메타 목표가 코어 루프를 N회 더 해야 한다는 동기를 주는가?
```

### 타워 디펜스

**핵심 경험**: 적의 경로를 예측하고 타워를 배치하여 막아내는 공간 전략의 재미

**핵심 경험 가설**: "타워를 어디에 놓을지 고민할 때 '이쪽에 놓으면 효율적이겠다' 전략이 나오면 핵심 재미가 있다"

**MVP 필수 기능**:

| 기능 | 플레이어 시나리오 | 핵심 수용 기준 |
|------|-----------------|--------------|
| 타워 배치 | 플레이어는 지정 가능 영역에 타워를 배치할 수 있다 | 적 경로 위에는 배치 불가, 배치 가능 영역 시각적으로 구분 |
| 타워 공격 (1~2종) | 배치된 타워는 사거리 내 적을 자동 공격한다 | 사거리 범위 표시 가능 (배치 시) |
| 적 경로 이동 (1종) | 적은 웨이포인트를 따라 이동한다 | 경로 이탈 없음, 경로가 시각적으로 표시됨 |
| 웨이브 시스템 | 일정 간격으로 다음 웨이브가 시작된다 | 웨이브 번호와 다음 웨이브 타이머 표시 |
| 골드 경제 | 적 처치 시 골드를 획득하고 타워 구매에 사용한다 | 골드 부족 시 구매 불가 + 시각적 안내 |
| 기지 HP | 적이 경로 끝에 도달하면 기지 HP가 감소하고 0이 되면 패배 | HP 감소 즉시 시각/사운드 피드백 |
| 웨이브 클리어 | 웨이브 내 모든 적을 처치하면 다음 웨이브로 진행한다 | 클리어 후 배치 시간(10~30초) 제공 |

**의도적 제외 (MVP)**:
- 맵 다양성 → 알파
- 타워 업그레이드 → 알파
- 보스 웨이브 → 알파
- 특수 능력 타워 → 알파

### 퍼즐 (규칙 발견형)

**핵심 경험**: 게임의 규칙을 스스로 발견하고 그것을 적용하는 "아하!" 순간의 지적 만족감

**핵심 경험 가설**: "설명 없이 첫 레벨에서 핵심 규칙을 스스로 파악하면 핵심 재미가 있다"

**MVP 필수 기능**:

| 기능 | 플레이어 시나리오 | 핵심 수용 기준 |
|------|-----------------|--------------|
| 핵심 메카닉 1가지 | 플레이어는 게임의 핵심 규칙을 적용하는 행동을 할 수 있다 | 규칙을 텍스트로 설명하지 않아도 1~2번의 시도로 파악 가능한 구조 |
| 즉각적 피드백 | 행동의 결과가 0.5초 이내에 명확히 표시된다 | 성공/실패 상태가 시각적으로 즉각 구분됨 |
| 레벨 목표 | 각 레벨에는 달성 조건이 있다 | 달성 조건이 화면에 항상 표시됨 |
| 레벨 클리어 판정 | 목표 달성 시 클리어 판정이 된다 | 클리어 피드백(이펙트/사운드)이 즉각 출력됨 |
| 난이도 순서 레벨 5~10개 | 첫 레벨은 규칙 학습, 마지막은 규칙 조합 | 중간 레벨 없이 직전 레벨 클리어만으로 다음 레벨 진행 가능 |
| 리셋 | 현재 레벨을 처음 상태로 되돌릴 수 있다 | 리셋 후 모든 변경 사항이 초기화됨 |

**의도적 제외 (MVP)**:
- 힌트 시스템 → 알파 (MVP는 레벨 설계로 학습 유도)
- 별 평가/시간 측정 → 알파
- 다중 메카닉 조합 → 알파

## 자주 하는 MVP 범위 과대 추정 실수

### 실수 1: "콘텐츠가 너무 부족하면 재미없을 것 같아"

**현실**: 핵심 경험이 없으면 콘텐츠 100개도 의미 없음. 핵심 경험이 있으면 1가지 콘텐츠로 검증 가능.

**올바른 질문**: "이 기능이 없으면 핵심 경험을 느낄 수 없는가?" → 아니라면 제외.

---

### 실수 2: "완성된 UI가 없으면 플레이어가 이해 못할 것 같아"

**현실**: 임시 UI(텍스트, 단색 버튼)로도 플레이 테스트 충분히 가능. 플레이어는 생각보다 잘 적응함.

**해결책**: MVP 수용 기준에 "UI 완성도"가 아닌 "플레이어가 3초 이내에 파악 가능"으로 기술.

---

### 실수 3: "세이브가 없으면 아무도 안 할 것 같아"

**현실**: MVP 플레이 테스트는 15~30분 단일 세션. 세이브 불필요.

**해결책**: MVP 수용 기준에 세이브 관련 항목 제외. 알파 단계에 추가.

---

### 실수 4: "메타 진행이 없으면 리플레이성이 없어"

**현실**: 코어 루프의 재미가 있으면 메타 없이도 재시작함. 메타는 장기 리텐션을 위한 것.

**해결책**: 핵심 경험 가설을 "자발적 재시작"으로 설정. 메타 없이 자발적 재시작이 나오면 성공.

---

### 실수 5: "좋은 그래픽이 없으면 인상이 안 좋을 것 같아"

**현실**: 플레이 테스트에서 그래픽은 핵심 경험 검증에 거의 영향 없음.

**해결책**: MVP 수용 기준에 그래픽 완성도 항목 없음. Unity 기본 도형 사용 허용.

---

## MVP 플레이 테스트 수용 기준

핵심 경험 가설 검증을 위한 플레이 테스트 기준.

| 지표 | MVP 합격 기준 | 비고 |
|------|-------------|------|
| 핵심 루프 파악 시간 | 설명 없이 5분 이내 | 플레이어가 스스로 파악 |
| 자발적 재시작률 | 테스터의 50% 이상 | 강제 재시작 제외 |
| 세션 지속 시간 | 계획 세션의 130% 이상 | 더 오래 하면 재미있음 신호 |
| "다시 하고 싶다" 응답률 | 60% 이상 | 플레이 직후 질문 |

**플레이 테스트 관찰 항목**:
```
□ 처음 30초 내 첫 행동: 무엇을 먼저 클릭/조작하는가?
□ 막히는 지점: 어디서 멈추는가?
□ 감탄/좌절 반응: 어느 순간에 발생하는가?
□ 재시작 여부: 자발적으로 재시작 버튼을 누르는가?
□ 질문 없이 진행 가능 여부: 조작 방법을 묻는가?
```

**플레이 테스트 후 질문**:
- "가장 재밌었던 순간은 언제였나요?"
- "다시 하고 싶나요? 이유는?"
- "답답하거나 이해 안 된 부분이 있었나요?"
- "이 게임의 목표가 무엇인 것 같나요?" (온보딩 검증)
