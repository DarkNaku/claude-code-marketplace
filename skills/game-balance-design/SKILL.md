# game-balance-design — 게임 밸런스 수치 설계

## 목적

PRD가 WHAT을 정의했다면, 밸런스 시트는 **숫자**를 정의합니다.

개발자가 구현에 착수하기 전에 필요한 초기 수치 파라미터를 제공합니다.

> 밸런스 수치는 플레이테스트의 **출발점**입니다. 정답이 아니라 검증해야 할 가설입니다.

---

## 입력 형태

다음을 입력받습니다:
- **PRD 문서**: `game-prd-writer` 출력 (기능 목록 + 수용 기준)
- **게임 분석 결과**: `game-analysis` 출력 (레퍼런스 데이터, 루프 타이밍)
- 또는 간단한 게임 설명과 장르

입력이 부족하면 다음을 질문합니다:
1. 게임 장르와 핵심 루프 (한 사이클에 얼마나 걸리는가?)
2. 세션 목표 길이 (한 세션에 몇 분?)
3. 타겟 난이도 (캐주얼 / 중간 / 하드코어)
4. 플레이어가 처음 성공 경험을 하기까지 허용되는 실패 횟수

---

## 수치 설계 워크플로우

### 1단계: 세션 길이 기준 역산

모든 수치는 **목표 세션 길이**에서 역산합니다.

```
목표 세션 길이 → 마이크로 루프 반복 횟수 → 한 루프당 허용 시간
→ 스폰율 / 공격 주기 / 능력치 성장 속도 기준 도출
```

**예시**:
```
목표 세션: 15분
마이크로 루프(전투 1회): 3~5초
→ 세션당 약 180~300회 마이크로 루프 발생
→ 이 횟수 동안 플레이어가 성장감을 느껴야 함
→ 레벨업 주기: 초반 10~15회, 후반 30~50회
```

### 2단계: 앵커 수치 결정

시스템의 기준이 되는 핵심 수치를 먼저 결정합니다.

| 시스템 | 앵커 수치 | 결정 기준 |
|--------|---------|---------|
| 전투 | 플레이어 기본 HP | "몇 번 맞으면 죽는가?" |
| 레벨업 | 레벨 1→2 소요 시간 | "첫 레벨업까지 몇 초?" |
| 경제 | 기본 자원 획득량 | "첫 구매/업그레이드까지 몇 루프?" |
| 스폰 | 초기 적 등장 간격 | "첫 5초는 얼마나 안전한가?" |

### 3단계: 파생 수치 계산

앵커 수치를 기반으로 나머지 수치를 비율로 결정합니다.

### 4단계: 난이도 곡선 수식

시간/레벨에 따른 수치 변화를 수식 또는 표로 표현합니다.

**일반적인 성장 공식 패턴**:
- **선형**: `수치 = 기준값 + (레벨 × 증가량)` — 단순하고 예측 가능
- **지수**: `수치 = 기준값 × 배율^레벨` — 후반 급격한 성장
- **단계별 고정**: 구간마다 수치 고정 — 밸런싱 쉬움, 직관적

### 수치 검증 시뮬레이션

설계한 수치로 핵심 시나리오를 언어 레벨에서 시뮬레이션합니다:

```
시나리오 1: 이상적인 플레이 (최적 선택)
  0:00 — 게임 시작. 플레이어 HP [수치], 적 0마리
  0:05 — 첫 적 등장, 처치. 경험치 [수치] 획득
  0:30 — 첫 레벨업. 스킬 선택.
  ...
  [N]분 — 세션 종료 예상 상태

시나리오 2: 어려운 플레이 (일부 실수)
  ...

문제 발견:
- [수치가 이상한 지점과 이유]
- [조정 권고안]
```

## 모범 사례

1. **세션 길이부터**: 모든 수치는 목표 세션 길이를 기준으로 역산
2. **앵커 먼저**: 전체 시스템에서 가장 중요한 수치 1~2개를 먼저 결정, 나머지는 비율로 도출
3. **단순한 공식**: 복잡한 공식보다 직관적인 선형 또는 단계별 공식 우선
4. **시뮬레이션 필수**: 수치 작성 후 언어 레벨로 "이 수치라면 [N]분에 무슨 일이 일어나는가?" 시뮬레이션
5. **가설 명시**: 각 수치 옆에 근거와 가설을 기록 — 플레이테스트 후 어느 수치를 왜 바꿨는지 추적 가능하게
6. **범위 표기**: 단일 수치보다 범위 `[최솟값 ~ 최댓값]`으로 표기하여 조정 여지 확보
