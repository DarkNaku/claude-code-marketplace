## Hades — 로그라이크 액션 RPG

### 2. MDA 분석

**에스테틱 (플레이어가 경험하는 감정)**

- **Challenge**: 매 런마다 달라지는 빌드와 적 배치에 적응하며 숙련도가 올라가는 쾌감
- **Narrative**: 사망 후 귀환할 때마다 새로운 대화가 펼쳐지고 NPC 관계가 실제로 변한다는 감각
- **Discovery**: 특정 신의 축복 조합이 시너지를 일으킬 때 "이런 조합이 있었나" 하는 발견의 재미

**다이나믹스 (에스테틱을 만드는 창발 행동)**

- 플레이어가 빌드 방향을 정하고 보상을 선택하면서, 예상치 못한 시너지가 발현되어 런마다 다른 강점의 빌드가 완성됨
- 사망이 페널티가 아닌 "스토리 진행 트리거"로 작동하여 실패 후 재도전 동기가 일반 로그라이크보다 강하게 유지됨
- 각 NPC와의 대화가 플레이 횟수에 따라 분기하며 플레이어가 "관계를 쌓고 있다"는 감각을 만듦

**메카닉 (다이나믹스를 구성하는 규칙)**

- 6종 무기 × 올림포스 신 12명의 축복 × 해머 업그레이드 조합 → 수천 가지 빌드 경우의 수
- 사망 시 하데스 궁전 귀환 → NPC별 대화 트리 진행 → 영구 자원(다크니스, 타이탄의 피) 획득 → 거울/키프샤케 업그레이드
- Heat 시스템: 클리어 후 자발적으로 난이도를 높이면 보상 배율 상승 → 하드코어 플레이어 리텐션 유지

### 3. 핵심 게임플레이 루프

**마이크로 루프** (3~15초, 한 전투 방):
```
방 진입 → 적 배치 파악 → 공격/회피/스킬 조합 → 적 전멸 → 보상 획득
```

**코어 루프** (5~15분, 한 지역 통과):
```
방 선택 (보상 미리보기) → 전투 → 보상 선택 (빌드 방향 결정) → 다음 방 → 보스전
```
→ 한 지역(타르타로스~스틱스 4개 지역 중 1개) 클리어가 코어 루프 1사이클

**매크로 루프** (30~60분, 런 1회):
```
탈출 시도 → 4개 지역 순차 진행 → 최종 탈출 성공 or 사망
    ↓
하데스 귀환 → NPC 대화 진행 → 영구 자원 소비 → 업그레이드 → 재도전
```

**메가 루프** (수십 시간, 장기 리텐션):
```
엔딩 1회 달성 → 진짜 엔딩(10회 탈출) 목표 → Heat 도전 → 모든 NPC 관계 완성 → 진 엔딩
```

**분석 포인트**:
- 플레이어 동기: 빌드 성공 카타르시스 + "다음 귀환에서 어떤 대화가 나올까" 하는 서사 기대
- 보상 타이밍: 방 클리어마다 보상(30~60초 간격), 귀환마다 대화(런 종료마다)
- 코어-메타 연결 강도: 강함 — 런 진행 중 수집한 자원이 메타 업그레이드에 직접 사용됨

### 4. 플레이어 동기 분석

| 동인 | 활용 | 게임 내 표현 방식 |
|------|------|------------------|
| Epic Meaning | ● | 신화 속 탈출 서사, 하데스와의 관계 |
| Development | ● | 거울 영구 업그레이드, 키프샤케 능력 |
| Creativity | ● | 빌드 조합 설계, 무기 플레이스타일 선택 |
| Ownership | ○ | 하우스 장식, 무기 외형 변경 (소폭) |
| Social Influence | ○ | 기록 공유, 온라인 빌드 토론 (간접) |
| Scarcity | ● | 런 내 보상 선택의 1회성, 캐릭터 호감도 아이템 희소 |
| Unpredictability | ● | 랜덤 축복/유물, 매번 다른 빌드 결과 |
| Loss Avoidance | ● | 사망 직전 포션 사용 판단, Heat 설정 신중 선택 |

**플레이어 여정 단계별 주요 동인**:
- Discovery: Narrative(신화 세계관 매력) + Challenge(첫 방 전투 직접 경험)
- Onboarding: Development(첫 귀환 후 거울 업그레이드 가능) + Unpredictability(어떤 축복이 나올까)
- Scaffolding: Creativity(빌드 조합 전략화) + Epic Meaning(스토리 진행, NPC 관계 심화)
- Endgame: Challenge(Heat 극한 도전) + Ownership(전체 NPC 관계 완성)

### 5. 플레이어 의사결정 구조

**경로/전략 선택**:
- 방 보상 미리보기에서 체력 회복 vs 필요한 축복 선택 → 현재 빌드 상태와 리스크 판단 필요

**자원 배분**:
- 캠프파이어: 회복 vs 카드 업그레이드 (체력이 낮으면 회복, 빌드가 불완전하면 업그레이드)
- 귀환 후 자원: 다크니스를 거울 어떤 능력에 투자할지

**리스크/보상 판단**:
- Heat 시스템: 클리어 후 자발적 난이도 추가 → 난이도 ↑, 보상 ↑, 실패 위험 ↑

**빌드 방향 결정**:
- 신의 축복 3개 중 1개 선택 → 현재 빌드와 시너지 있는 신 vs 즉시 강한 능력
- 듀오 축복(두 신의 조합 효과) 노리기 vs 안전하게 단독 강화

### 6. 게임플레이 시스템 분해

**전투 시스템**
- 역할: 마이크로 루프 핵심, 에스테틱 Challenge의 원천
- 입력: 방향키+공격/특수/대시/콜 입력
- 출력: 적 HP 감소, 자원 획득, 방 클리어
- 주요 파라미터: 무기별 공격 속도·범위·데미지, 대시 무적 프레임(i-frame)

**빌드 시스템 (신의 축복)**
- 역할: 매크로 루프마다 빌드 다양성 보장, Discovery 에스테틱 생성
- 입력: 전투 보상 선택(3개 중 1개)
- 출력: 공격/스킬/대시에 신 고유 효과 부여, 듀오 조건 충족 시 시너지 발동
- 주요 파라미터: 각 신의 축복 등급(일반~레전더리), 듀오 조건(특정 두 신의 축복 동시 보유)

**메타 진행 시스템**
- 역할: 매크로 루프 보상 소비처, 장기 리텐션 담당
- 입력: 런에서 수집한 다크니스·타이탄의 피·다이아몬드·암브로시아
- 출력: 거울(영구 능력치 선택형 업그레이드), 키프샤케(패시브 능력), NPC 호감도 상승
- 주요 파라미터: 거울 양면 선택(A 능력 vs B 능력), 호감도별 대화 분기 수

**내러티브 시스템**
- 역할: 메가 루프 동기 제공, Narrative 에스테틱 전담
- 입력: 런 횟수, 사망/성공, NPC와의 선물 교환 횟수
- 출력: NPC별 신규 대화 언락, 관계 스코어 상승, 서사 챕터 진행
- 주요 파라미터: NPC별 대화 트리 깊이(가장 많은 제우스 기준 수십 개 분기)

### 7. 보상 구조

**즉각 보상** (마이크로 루프 완결):
- 적 처치 시 이펙트+사운드+황금 드롭 → 스킬 아톰 피드백 즉각 완결
- 크리티컬 시 일시적 슬로우모션 → 고급 피드백

**단기 보상** (코어 루프 완결):
- 지역 보스 처치 → 강력한 유물 또는 희귀 축복
- 방 클리어마다 보상 선택(30~60초 주기)

**장기 보상** (매크로/메가 루프):
- 런 완료 후 귀환: 반드시 새로운 NPC 대화 1회 이상
- 첫 탈출 성공(10~15시간): 게임 전체를 뒤집는 반전 스토리 공개
- NPC 관계 최대치: 로맨스 라인, 진 엔딩

**변수 보상 설계**:
- 랜덤 3개 축복 제시: 원하는 축복이 나올 수도, 안 나올 수도 있음 → 호기심 유지
- 방 보상 구성도 런마다 다름 → Fixed(귀환 시 대화)와 Variable(런 내 빌드)의 조합

**보상 빈도 분석**:
- 방 클리어 보상: 약 1~2분마다
- 귀환 대화(새 내용): 런 1회 종료마다 (30~60분)
- 스토리 분기점: 약 5~10런마다 주요 전개

### 8. 난이도 곡선

**난이도 증가 방식**:
- 지역별 적 강화: 타르타로스(입문) → 스틱스(숙련 요구)
- 보스 4명 각각 고유 패턴 — 처음엔 패턴 파악에 여러 번 사망

**플레이어 성장 속도**:
- 초반(0~5런): 메타 업그레이드로 빠른 성장 체감
- 중반(5~20런): 빌드 이해도 상승, 첫 탈출 달성
- 후반(20런+): Heat 시스템으로 자발적 난이도 상승, 마스터리 단계

**주요 난이도 분기점**:
- 3번째 지역 엘리시움 보스(테세우스+아스테리오스): 2명 동시 전투 → 대부분의 첫 이탈 발생
- Heat 32 클리어: 커뮤니티 인정 마스터리 기준

**이탈 위험 구간**:
- 0~3런: 첫 탈출 실패의 좌절 → 귀환 대화와 빠른 업그레이드가 완충
- 엘리시움 보스 반복 사망 → "거울 업그레이드를 더 모으면 클리어 가능"이라는 진행 신호 필요

### 9. UX / 피드백 디자인

**핵심 액션 피드백**:
- 무기별 차별화된 타격음과 이펙트(검 = 금속음, 권총 = 폭발음)
- 크리티컬 시 0.1초 슬로우모션 → 스킬 아톰 피드백의 절정
- 대시 무적 프레임에 시각적 잔상 → "내가 피했다"는 명확한 인지

**정보 전달**:
- 적 공격 전 바닥에 경고 표시(레드존) → 인지 부하 분산
- 보상 창에 축복 효과 미리보기 → 의사결정 지원
- 체력이 임계값 이하: 화면 가장자리 붉게 표시

**온보딩/튜토리얼 구조**:
- 첫 런: 잠금 해제된 메카닉만 등장, 자동 튜토리얼 팝업 최소화
- 새 능력 첫 획득 시 자동 설명 → 학습 부담을 플레이 맥락 안에 배치
- 사망 후 귀환 = 자연스러운 숙고 시간 → 실패 원인 분석 유도

**인지 부하 관리**:
- 한 번에 3개 축복만 선택지로 제시 → 결정 피로 방지
- 현재 빌드 요약을 보상 화면에 표시 → 선택 맥락 유지

### 10. 콘텐츠 구조

**콘텐츠 단위**: 방(Room) — 전투 1회 또는 이벤트 1회
- 단위당 평균 플레이: 1~3분
- 런 1회 방 수: 약 20~30개

**콘텐츠 다양성 확보**:
- 절차적 방 배치: 같은 지역도 매번 다른 방 연결
- 올림포스 신 12명 × 무기 6종 조합 → 빌드 재현 불가

**콘텐츠 언락 구조**:
- 선형적 서사 진행(NPC 대화 트리) + 개방형 빌드 탐구의 혼합
- 특정 NPC 호감도 달성 시 무기 형태 변경(아스펙트) 언락

### 11. 경제 및 수익화 구조

**자원 종류**:
- 다크니스(런 내 드롭): 거울 업그레이드 재료 — 가장 기본적인 메타 화폐
- 타이탄의 피(보스 첫 처치): 무기 아스펙트 언락 — 희소, 장기 목표
- 암브로시아(관계 보상): NPC 관계 아이템 — 사회적 진행의 상징
- 골드(런 내): 상점에서 사용, 런 종료 시 소멸

**경제 밸런싱**:
- 다크니스는 빠르게 공급 → 초반 메타 진행 가속감 제공
- 타이탄의 피는 보스 최초 처치에만 획득 → 새로운 무기 탐구 동기
- 골드는 런 내에서만 유효 → 소비 결정에 긴장감

**수익화**:
- 일회성 패키지 구매 (DLC 없음, 가챠 없음)
- PC/콘솔 기준 수익화 접점이 코어 경험과 완전히 분리됨

### 12. 디자인 기둥

1. **실패도 진행**: 사망이 패널티가 아니라 스토리 트리거 → 로그라이크의 좌절감 최소화
2. **빌드 다양성**: 매 런마다 다른 빌드 → Challenge + Discovery 에스테틱의 원천
3. **내러티브 통합**: 게임플레이 행동이 스토리에 직접 반영 → 서사와 루프의 분리 없음
4. **점진적 마스터리**: Heat 시스템으로 숙련 플레이어에게 지속적 도전 제공
5. **즉각적 피드백**: 모든 행동에 명확한 시청각 피드백 → 스킬 아톰 완결

---

### 13. 재사용 가능한 디자인 패턴

**패턴 1: 사망을 진행 트리거로 전환**
- 설명: 런 실패(사망)를 서사 이벤트로 연결하여 리플레이 동기를 유지
- 효과: 로그라이크의 주요 이탈 원인(반복 좌절)을 서사 호기심으로 상쇄
- 예시: 귀환마다 하데스와의 새 대화, NPC 관계 점수 진행

**패턴 2: 선택 제한이 전략성 강화**
- 설명: 매 보상마다 3개 중 1개만 선택 → 제한이 오히려 빌드 전략화 강제
- 효과: 무한 선택지보다 의미 있는 의사결정 밀도 높임
- 예시: 랜덤 3개 축복 제시, 캠프파이어 회복/업그레이드 택일

**패턴 3: 자발적 난이도 조절 (Heat 시스템)**
- 설명: 클리어 후 플레이어 스스로 난이도 조건을 추가하는 레이어 제공
- 효과: 숙련 플레이어 이탈 방지, 난이도 상승과 보상 상승을 연동
- 예시: Heat 1~32개 조합, Heat 조합마다 다른 전략 필요

---

### 14. 프로토타입 범위 추천

**MVP**:
- 플레이어 기본 공격 + 특수 공격 + 대시 (스킬 아톰 코어 검증)
- 단순 적 2종 + 간단한 패턴
- 방 5개 연속 진행 + 마지막 보스 1명
- 신의 축복 5개 (단순 수치 증가)
- 사망 후 재시작 (내러티브 없이 루프만 검증)
- **검증 가설**: "빌드 선택만으로 매 런이 다르게 느껴지는가?"

**알파**:
- 무기 2종, 축복 20종, 시너지 3가지
- 지역 2개 + 보스 2명
- 메타 진행(거울 업그레이드) 기본 구현
- 귀환 시 간단한 대화 시스템

**베타**:
- 무기 6종, 축복 60종+, 듀오 시너지
- 전체 4지역 + 보스 4명
- NPC 6명 + 대화 분기
- Heat 시스템 + 진 엔딩 조건

## Royal Match — 하이브리드 캐주얼 (퍼즐 + 왕국 건설)

### 1. 게임 개요

```
게임: Royal Match
장르: 하이브리드 캐주얼 (3매치 퍼즐 + 왕국 건설 메타)
플랫폼: iOS, Android
개발사: Dream Games
출시일: 2021년 3월
타겟 유저: 퍼즐 장르 경험자, 빌더 게임 팬, 25~45세 여성 주요층

핵심 에스테틱 (LeBlanc 8유형 중):
  Challenge(레벨 클리어 도전), Ownership(왕국 소유/건설), Narrative(왕국 복원 서사)

게임 경험 요약:
  제한된 이동 수 안에 퍼즐을 클리어하면 얻은 별로 왕국을 직접 꾸미는 —
  도전의 성취감과 내 것을 만드는 소유감을 동시에 제공하는 하이브리드 캐주얼

조사 출처:
  - https://medium.com/@ekinmelissezer/game-analysis-for-royal-match-and-toon-blast-9c4bff8ef48b — 게임 분석
  - https://www.gameanalytics.com/blog/six-games-that-successfully-layer-in-meta-mechanics — 메타 레이어 분석
  - https://store.steampowered.com 외 모바일 마켓 정보
```

---

### 2. MDA 분석

**에스테틱 (플레이어가 경험하는 감정)**

- **Challenge**: "이동 수가 몇 개 남지 않았는데 클리어할 수 있을까" — 퍼즐의 긴장과 성취
- **Ownership**: 클리어한 별로 왕국의 방을 직접 변화시키는 "내 왕국" 감각
- **Narrative**: 킹덤 복원 스토리, 방마다 왕족 캐릭터들의 이야기

**다이나믹스 (에스테틱을 만드는 창발 행동)**

- 제한된 이동 수 안에서 연쇄 매칭(콤보)을 노리면 파워업이 폭발적으로 발동 → Challenge 절정
- 퍼즐 클리어 별이 왕국 건설에 즉시 소비됨 → 퍼즐 → 시각적 변화가 매우 짧은 피드백 루프로 연결
- 방마다 배경 캐릭터가 반응 → 단순 꾸미기가 아닌 "이야기 속 공간 복원" 감각

**메카닉 (다이나믹스를 구성하는 규칙)**

- 3매치 이상 연결 시 제거, 5매치→파워업, 연속 파워업→연쇄 폭발
- 이동 수(Moves) 고정 제한 — 목표(특정 타일 제거, 장애물 제거 등)를 이동 수 내에 달성
- 클리어 시 별 1~3개 획득 (남은 이동 수 기반), 별을 왕국 건설 미션 완료에 소비

---

### 3. 핵심 게임플레이 루프

**마이크로 루프** (3~10초, 한 번의 매칭):
```
보드 파악 → 3개 이상 연결 경로 탐색 → 스와이프 → 타일 제거 + 연쇄 확인 → 다음 수 계획
```

**코어 루프** (2~5분, 레벨 1개):
```
레벨 목표 확인 → 이동 수 소비하며 퍼즐 진행 → 클리어 or 실패
    → 클리어: 별 1~3개 획득
    → 실패: 생명(Heart) 1 소모, 30분 대기 or 구매
```

**매크로 루프** (10~20분, 왕국 건설 진행):
```
별 누적 → 왕국 미션 달성 (방 1개 완성) → 다음 미션 열람 → 새 레벨 팩 해금
```

**메가 루프** (수일~수주, 장기 리텐션):
```
왕국 챕터 완성 → 새 지역 해금 → 이벤트 참여(한정 보상) → 다음 챕터 대기
```

**분석 포인트**:
- 코어-메타 연결 강도: **매우 강함** — 코어(퍼즐)의 보상인 별이 메타(왕국 건설)의 유일한 화폐
- 이 단일 화폐 구조가 "퍼즐을 클리어하면 왕국이 바뀐다"는 직접 인과관계를 만듦

---

### 4. 플레이어 동기 분석

| 동인 | 활용 | 게임 내 표현 방식 |
|------|------|------------------|
| Epic Meaning | ● | 왕국 복원 서사, 왕족 캐릭터 스토리 |
| Development | ● | 레벨 숫자 증가, 왕국 시각 변화 |
| Creativity | ○ | 방 꾸미기 선택지(제한적) |
| Ownership | ● | "내 왕국" 건물이 실제로 변화 |
| Social Influence | ● | 친구 리더보드, 왕국 방문 기능 |
| Scarcity | ● | 생명(Heart) 5개 제한, 30분 재생 |
| Unpredictability | ● | 보드 타일 랜덤 배치, Near Miss 설계 |
| Loss Avoidance | ● | "이동 수 3개면 부스터 쓸까" 판단, 생명 소모 회피 |

**플레이어 여정 단계별 주요 동인**:
- Discovery: Challenge(첫 레벨의 간단한 퍼즐 성취) + Narrative(왕국 소개 컷씬)
- Onboarding: Development(빠른 레벨업 + 왕국 첫 변화) + Ownership(내 방이 생겼다)
- Scaffolding: Challenge(난이도 상승 레벨 도전) + Scarcity(생명이 없어서 나중에 다시)
- Endgame: Social(친구 랭킹) + Ownership(왕국 완성) + Scarcity(한정 이벤트)

---

### 5. 플레이어 의사결정 구조

**보드 전략 판단**:
- 어디를 먼저 깰 것인가: 목표 타일 직접 공략 vs 파워업 축적 후 폭발
- 파워업 조합: 로켓+로켓 = 십자 폭발, 로켓+폭탄 = 대형 폭발 → 조합 타이밍 판단

**리스크/보상 판단**:
- 이동 수 3개 이하 남았을 때: 부스터 소비(유료 화폐) vs 실패하고 재시도
- Near Miss 상황: "한 번만 더면 될 것 같다" → 생명 구매 충동 유발

**소비 판단**:
- 보상형 광고 시청 vs 건너뛰기
- 유료 이동 수 추가(5회) 구매 여부

---

### 6. 게임플레이 시스템 분해

**퍼즐 엔진**
- 역할: 코어 루프 핵심, Challenge 에스테틱 생성
- 입력: 스와이프 방향
- 출력: 타일 제거 수, 연쇄 발동, 이동 수 감소, 목표 달성 여부
- 주요 파라미터: 보드 크기(주로 9×9), 이동 수(15~35), 목표 타일 수, 장애물 유형

**파워업 시스템**
- 역할: Challenge의 절정(폭발 카타르시스)과 전략적 깊이 제공
- 입력: 매칭 수(5개=로켓, 6개=폭탄, T자=프로펠러)
- 출력: 넓은 범위 타일 제거
- 주요 파라미터: 파워업 조합 효과(2개 조합 시 시너지 폭발)

**생명 시스템 (Heart)**
- 역할: 세션 길이 조절, 수익화 접점
- 입력: 레벨 실패
- 출력: 생명 1 감소, 0개 시 플레이 불가
- 주요 파라미터: 최대 5개, 30분마다 1개 재생, 친구 선물로 1개 추가

**왕국 건설 시스템 (메타 레이어)**
- 역할: 매크로 루프 보상 소비처, Ownership/Narrative 에스테틱 담당
- 입력: 별 소비
- 출력: 방 내 오브젝트 시각 변화, 캐릭터 대사 출력, 다음 미션 공개
- 주요 파라미터: 미션당 별 1~3개 소비, 방 완성에 미션 5~10개 필요

**레벨 진행 시스템**
- 역할: 코어 루프 지속 제공
- 주요 파라미터: 레벨 수 수천 개 이상, 난이도 점진 상승, 주기적 이지-하드-이지 사이클

---

### 7. 보상 구조

**즉각 보상** (마이크로 루프):
- 매칭 성공 시 경쾌한 팝 사운드 + 타일 소멸 애니메이션
- 연쇄 발동 시 폭발 이펙트 확대 → 콤보 피드백 절정

**단기 보상** (코어 루프):
- 레벨 클리어 시 별 획득 애니메이션 + 남은 이동 수 소비 연출
- 왕국 미션 달성 시 방 변화 컷씬

**장기 보상** (매크로/메가):
- 왕국 챕터 완성 시 스토리 컷씬
- 이벤트 한정 장식 아이템

**변수 보상 설계**:
- 보드 타일 배치 랜덤 → 매 레벨 초반 보드 상태가 다름
- Near Miss 설계: 이동 수 1~2개 남기고 실패 → "조금만 더"라는 심리 최대화

**보상 빈도 분석**:
- 파워업 생성: 레벨당 평균 3~5회 (15~30초마다)
- 레벨 클리어: 평균 2~4분 (초반), 10분+ (고난도)
- 왕국 미션 완료: 레벨 1~3개 당 1회

---

### 8. 난이도 곡선

**난이도 증가 방식**:
- 장애물 유형 도입: 얼음(2회 타격), 체인(1회 타격), 상자(열쇠 필요) 순차 추가
- 이동 수 제한 감소: 초반 30이동 → 중반 20이동 → 고난도 15이동 이하
- 목표 복합화: 단일 목표 → 동시 다중 목표

**플레이어 성장 속도**:
- 초반 1~50레벨: 쉬운 레벨 연속 → 장르 입문자 온보딩
- 중반 50~200레벨: 이지-하드 교차 패턴 정착
- 후반 200레벨+: 레벨당 평균 3회 이상 시도가 정상적인 난이도

**이탈 위험 구간**:
- 생명 5개 소진 시: 30분 대기 or 구매 갈림길
- 연속 실패 3회 이상: 부스터 제안 팝업 등장 → 완충 장치

---

### 9. UX / 피드백 디자인

**핵심 액션 피드백**:
- 스와이프 → 즉각 타일 이동 + 경쾌한 팝 사운드 → 스킬 아톰 완결이 0.3초 이내
- 파워업 폭발: 화면 전체를 채우는 이펙트 + 강한 사운드 → 절정 피드백

**정보 전달**:
- 남은 이동 수: 화면 중앙 상단 대형 숫자 표시
- 목표 진행률: 레벨 목표 아이콘에 달성 수/전체 수 표시
- 생명 수: 상단 고정 표시

**온보딩**:
- 첫 10레벨: 필요한 스와이프 방향을 손가락 애니메이션으로 직접 안내 (명시적 튜토리얼)
- 파워업 첫 생성 시 자동 팝업 설명
- 왕국 건설 첫 미션: 전체 UI를 흐리고 버튼 강조

**인지 부하 관리**:
- 보드 9×9 고정(변형 없음) → 공간 파악 부담 없음
- 목표는 2~3가지 이하로 동시 제시

---

### 10. 콘텐츠 구조

**콘텐츠 단위**: 레벨(Level)
- 레벨당 평균 플레이: 2~5분
- 챕터당 레벨 수: 약 15~25개

**콘텐츠 다양성 확보**:
- 수동 제작 레벨(핸드크래프트) — 보드 배치 고정, 반복 시 같은 보드
- 주기적 신규 장애물 유형 추가로 신선도 유지

**콘텐츠 언락 구조**:
- 순차 선형 진행: 이전 레벨 클리어 → 다음 레벨 해금
- 왕국 챕터가 레벨 팩과 연동: 챕터 완성 → 새 왕국 지역 + 새 레벨 팩 동시 오픈

---

### 11. 경제 및 수익화 구조

**자원 종류**:
- 별(Star): 퍼즐 클리어 보상, 왕국 건설 유일 화폐 — 코어-메타 연결 핵심
- 생명(Heart): 시도권, 30분마다 1개 재생, 최대 5개
- 코인: 소프트 화폐, 부스터 구매 가능
- 로얄 골드: 유료 화폐, 코인/생명/이동 수 구매

**자원 획득 경로**:
- 별: 레벨 클리어 시 1~3개 (이동 수 여유에 따라)
- 생명: 시간 재생 / 친구 선물 / 광고 시청(1개) / 유료 구매
- 코인: 레벨 클리어 보너스, 이벤트 보상

**수익화 접점**:
- 보상형 광고: 추가 이동 수 3개 (실패 후 자발적), 생명 1개
- 인터스티셜 광고: 레벨 클리어 후 간헐적 등장
- 인앱 결제: 생명 팩, 코인 팩, 부스터 팩
- **핵심**: 수익화가 "실패 후 계속하고 싶은 욕구" 시점에 집중 → Loss Avoidance 동인 활용

**에너지 시스템**:
- 생명 5개, 실패 시 1개 소모, 30분마다 1개 재생
- 세션 길이 자연 제한: 연속 플레이 최대 약 15~20레벨 → 중독성 조절

**경제 밸런싱**:
- 별은 플레이 외 획득 불가(광고/구매 우회 없음) → 코어 루프 반드시 플레이하게 설계
- 생명은 유료 구매 가능 → 수익화 접점, 단 무료 재생으로 이탈 방지

---

### 12. 디자인 기둥

1. **단일 화폐로 코어-메타 직결**: 별 1개가 퍼즐 성취 + 왕국 진행을 동시에 의미
2. **소유감의 시각화**: 내가 클리어할 때마다 왕국이 바뀐다 → 공허한 숫자가 아닌 공간의 변화
3. **제한이 만드는 긴장감**: 이동 수 제한이 "이번엔 아슬아슬하게 됐다"는 도전의 핵심
4. **접근성 우선**: 규칙은 단순(스와이프), 깊이는 콤보와 파워업 조합에서
5. **서사로 방향 제시**: 왕국 스토리가 다음 레벨을 해야 하는 이유를 제공

---

### 13. 재사용 가능한 디자인 패턴

**패턴 1: 단일 화폐 코어-메타 연결**
- 설명: 코어 루프(퍼즐)의 보상이 메타 루프(건설)의 유일한 소비 화폐
- 효과: "퍼즐을 클리어 = 왕국이 성장한다"는 직접 인과 → 코어 플레이 동기 강화
- 예시: Royal Match의 별, 유사 사례 Township의 코인

**패턴 2: Near Miss 설계로 재시도 유도**
- 설명: 이동 수 1~2개 남기고 실패하도록 레벨 밸런싱 → "조금만 더"라는 감각
- 효과: 좌절 최소화 + 재시도 충동 최대화 → 수익화 접점(이동 수 구매)과 연동
- 예시: 레벨 클리어 직전 이동 수 소진 경험

**패턴 3: 이지-하드-이지 난이도 사이클**
- 설명: 쉬운 레벨 → 어려운 레벨 → 쉬운 레벨 반복으로 이탈 방지
- 효과: 어려운 레벨에서 이탈할 플레이어를 이후 쉬운 레벨로 붙잡음
- 예시: Candy Crush, Royal Match 모두 채택

---

### 14. 프로토타입 범위 추천

**MVP**:
- 3매치 퍼즐 엔진 (스와이프, 중력, 보충)
- 이동 수 제한 + 단일 목표(특정 타일 X개 제거)
- 파워업 1종 (로켓)
- 레벨 10개 수동 제작
- **검증 가설**: "이동 수 제한이 긴장감을 만드는가? 파워업이 카타르시스를 주는가?"

**알파**:
- 파워업 3종 + 조합 효과
- 장애물 2종
- 레벨 50개
- 왕국 건설 메타 기초 (별 → 방 1개 변화)
- 생명 시스템

**베타**:
- 파워업 5종 + 모든 조합
- 장애물 6종
- 레벨 200개+
- 왕국 챕터 3개 (각 방 5개 이상)
- 이벤트 시스템 기초
- 광고 수익화 연동

## Merge Mansion — 하이브리드 캐주얼 (머지 + 서사)

### 1. 게임 개요

```
게임: Merge Mansion
장르: 하이브리드 캐주얼 (오브젝트 머지 + 저택 복원 서사)
플랫폼: iOS, Android
개발사: Metacore Games
출시일: 2020년 11월
타겟 유저: 캐주얼 모바일 게이머, 25~45세 여성, 수집/정리 좋아하는 유저

핵심 에스테틱 (LeBlanc 8유형 중):
  Discovery(병합으로 새 오브젝트 발견), Narrative(가족 미스터리 서사), Ownership(저택 직접 복원)

게임 경험 요약:
  보드 위 오브젝트를 병합하여 미션 재료를 만들고,
  그것으로 낡은 저택을 복원하면서 가족의 숨겨진 비밀을 밝혀가는 서사 머지 게임

조사 출처:
  - https://www.linkedin.com/pulse/merge-mansion-game-analysis-1-daniele-siena — 상세 게임 분석
  - https://www.blog.udonis.co/mobile-marketing/mobile-games/merge-games-market — 머지 게임 시장 분석
  - https://www.gameanalytics.com/blog/six-games-that-successfully-layer-in-meta-mechanics — 메타 레이어 분석
```

---

### 2. MDA 분석

**에스테틱 (플레이어가 경험하는 감정)**

- **Discovery**: 두 오브젝트가 병합되어 새로운 오브젝트가 탄생하는 "이게 나오는구나" 발견의 연속
- **Narrative**: 할머니 에블린의 과거와 가족 비밀이 한 챕터씩 밝혀지는 미스터리 서사
- **Ownership**: 황폐한 저택이 내가 모은 재료로 복원되어 아름답게 변하는 공간 소유감

**다이나믹스 (에스테틱을 만드는 창발 행동)**

- 보드 위 오브젝트의 병합 트리를 파악하고 필요한 재료를 거꾸로 추적하는 인벤토리 관리 행동 발현
- 다음 서사 챕터를 보려면 미션을 완료해야 함 → 병합 행동이 서사 진행과 연결되어 "다음 이야기가 보고 싶다" 동기 강화
- 에너지 제한으로 즉각 소진 불가 → 보드 오브젝트 조합을 미리 계획하는 전략 행동

**메카닉 (다이나믹스를 구성하는 규칙)**

- 같은 오브젝트 2개 → 드래그 앤 드롭 병합 → 상위 오브젝트 1개 생성
- 에너지 소비로 보드에 새 오브젝트 소환, 에너지 시간 재생(최대 30개, 분당 1개)
- 미션: 특정 오브젝트 N개 제출 → 저택 특정 공간 복원 → 서사 대화 출력

---

### 3. 핵심 게임플레이 루프

**마이크로 루프** (5~15초, 병합 1회):
```
보드 스캔 → 같은 오브젝트 2개 확인 → 드래그 병합 → 새 오브젝트 확인 → 다음 병합 탐색
```

**코어 루프** (3~10분, 에너지 세션):
```
에너지 소비 → 오브젝트 소환 → 병합 연속 진행 → 미션 재료 확보 → 에너지 소진 or 미션 완료
```

**매크로 루프** (20~40분, 저택 복원 1개 공간):
```
미션 재료 병합 완성 → 제출 → 저택 공간 변화 + 서사 대화 → 다음 미션 공개 → 에너지 재충전 대기
```

**메가 루프** (수일~수주):
```
챕터(여러 공간 완성) 완료 → 서사 주요 반전 공개 → 새 지역(보일러실, 정원 등) 해금 → 계속 플레이
```

**분석 포인트**:
- 에너지 시스템이 세션 길이를 자연스럽게 조절 → 하루 2~3회 접속 패턴 유도
- 서사 드리블링(Drip-feeding): 미션 완료마다 스토리 조각을 조금씩 공개 → 다음 챕터 기대감 유지

---

### 4. 플레이어 동기 분석

| 동인 | 활용 | 게임 내 표현 방식 |
|------|------|------------------|
| Epic Meaning | ● | 가족 미스터리 서사, 저택의 역사 |
| Development | ● | 병합 트리 진행, 저택 복원 완성도 |
| Creativity | ○ | 인벤토리 배치 자유도 (제한적) |
| Ownership | ● | "내가 복원한 저택" 시각적 변화 |
| Social Influence | ○ | 일부 소셜 기능 (제한적) |
| Scarcity | ● | 에너지 제한, 보드 칸 수 한계 |
| Unpredictability | ● | 병합 결과 기대감, 이벤트 한정 오브젝트 |
| Loss Avoidance | ● | 보드가 꽉 찼을 때 스트레스 → 정리 욕구 |

**플레이어 여정 단계별 주요 동인**:
- Discovery: Discovery(첫 병합 결과 발견) + Narrative(할머니의 수상한 저택 소개)
- Onboarding: Development(빠른 미션 완료 + 저택 첫 변화) + Ownership(내 보드가 생겼다)
- Scaffolding: Narrative(스토리 반전 공개) + Scarcity(에너지 부족, 보드 꽉 참)
- Endgame: Epic Meaning(전체 미스터리 해명) + Ownership(모든 공간 복원)

---

### 5. 플레이어 의사결정 구조

**인벤토리 관리 판단**:
- 어떤 오브젝트를 지금 병합할지, 어떤 것을 보유할지 → 미션 목표를 보고 거꾸로 계획
- 보드 공간 부족 시: 낮은 레벨 오브젝트 병합으로 공간 확보 vs 현 상태 유지

**에너지 소비 판단**:
- 어떤 오브젝트를 소환할지 → 현재 미션 필요 재료 확인 후 역산

**진행 속도 판단**:
- 에너지 자연 재생 대기 vs 광고 시청으로 즉시 충전 vs 유료 구매

---

### 6. 게임플레이 시스템 분해

**병합 엔진 (코어 루프 핵심)**
- 역할: 마이크로 루프 핵심, Discovery 에스테틱 생성
- 입력: 드래그 앤 드롭 (같은 오브젝트 2개 겹치기)
- 출력: 상위 레벨 오브젝트 1개 생성, 보드 공간 1칸 회수
- 주요 파라미터: 병합 트리 깊이(오브젝트 레벨 1~10+), 보드 크기(약 5×9)

**에너지 시스템**
- 역할: 세션 길이 조절, 수익화 접점
- 입력: 오브젝트 소환 명령
- 출력: 에너지 소모, 새 오브젝트 보드 배치
- 주요 파라미터: 최대 30개, 분당 1개 재생, 소환 비용 오브젝트 레벨에 따라 1~5개

**미션 시스템 (코어-메타 연결)**
- 역할: 병합 행동에 목적을 부여, 서사 진행 트리거
- 입력: 특정 오브젝트 N개 제출
- 출력: 저택 공간 시각 변화, 서사 대화 출력, 다음 미션 공개
- 주요 파라미터: 미션당 필요 오브젝트 수(1~10개), 미션 체인 길이(공간 완성까지 5~15개)

**서사 시스템 (메가 루프 동기)**
- 역할: Narrative 에스테틱 담당, 장기 리텐션 핵심 동인
- 입력: 미션 완료 수
- 출력: 에블린/가족 대화 출력, 챕터 진행, 반전 공개
- 주요 파라미터: 챕터당 미션 수(약 10~20개), 주요 서사 반전 간격

---

### 7. 보상 구조

**즉각 보상** (마이크로 루프):
- 병합 성공 시 작은 팝 이펙트 + "새 오브젝트!" 텍스트
- 높은 레벨 오브젝트일수록 연출 크기 증가 → 병합 트리 상단 도달의 카타르시스

**단기 보상** (코어 루프):
- 미션 재료 완성 → 제출 버튼 활성화 + 시각적 강조

**장기 보상** (매크로/메가 루프):
- 저택 공간 복원 시 Before/After 애니메이션 → Ownership 에스테틱 절정
- 챕터 완성 시 서사 주요 반전 공개 (유일한 메가 루프 보상)

**변수 보상 설계**:
- 병합 결과의 "다음 레벨이 뭐가 나올까" 기대감 — Fixed(병합 결과는 고정)이나 처음엔 모름
- 이벤트 한정 오브젝트 등장 → Scarcity + Unpredictability 조합

**보상 빈도 분석**:
- 병합 1회: 5~15초마다 (에너지 있을 때)
- 미션 완료: 평균 5~15분
- 저택 공간 복원: 20~40분
- 서사 챕터 완성: 수일 (에너지 제한으로 자연스럽게 늘어남)

---

### 8. 난이도 곡선

**난이도 증가 방식**:
- 미션 필요 오브젝트 레벨 상승 → 더 많은 병합 단계 필요
- 보드 공간 관리 복잡도 증가 → 높은 레벨 오브젝트가 차지하는 칸이 늘어남

**플레이어 성장 속도**:
- 초반: 낮은 레벨 오브젝트만 필요 → 빠른 미션 완료 → 서사 빠르게 진행
- 중반: 에너지 제한이 실질적 병목이 됨 → 하루 2~3회 접속 패턴 정착
- 후반: 레벨 10+ 오브젝트 필요 미션 → 수일 단위 목표로 전환

**이탈 위험 구간**:
- 보드가 꽉 차서 병합 불가 상태: 어디를 병합해야 할지 몰라 답답함 → 첫 이탈 원인
- 에너지 소진 후 대기 시간: 다른 게임으로 이탈 위험 → 서사 기대감이 완충

---

### 9. UX / 피드백 디자인

**핵심 액션 피드백**:
- 드래그 시 대상 칸 하이라이트 → 병합 가능 여부 즉각 시각 확인
- 병합 성공 시 새 오브젝트 등장 애니메이션 + 레벨 수치 강조

**정보 전달**:
- 미션 목표: 항상 화면 측면에 필요 오브젝트 아이콘으로 고정 표시
- 에너지: 상단 고정, 재충전 타이머 표시
- 보드 잔여 공간: 암묵적(직접 보드를 보고 파악)

**온보딩**:
- 첫 5분: 병합 방법, 에너지 소환 방법, 미션 제출 순서를 화살표로 안내 (명시적)
- 서사 캐릭터(할머니 에블린)가 튜토리얼을 진행 → 서사 몰입과 학습 동시 진행

**인지 부하 관리**:
- 보드 크기 고정 → 공간 파악 익숙해지기 용이
- 병합 결과를 오브젝트 위 화살표 아이콘으로 힌트 제공 → 다음 단계 인지 지원

---

### 10. 콘텐츠 구조

**콘텐츠 단위**: 저택 공간(Room) + 미션 체인
- 공간 1개 완성에 미션 5~15개, 소요 시간 수시간~수일

**콘텐츠 다양성 확보**:
- 수동 제작 미션 체인 — 공간마다 다른 오브젝트 카테고리 사용 (정원=식물류, 주방=조리도구류)
- 신규 지역 추가로 새 병합 트리 도입 → 장기 플레이어 신선도 유지

**콘텐츠 언락 구조**:
- 순차 선형: 이전 공간 완성 → 다음 공간 해금
- 서사 드리블링: 공간 완성마다 이야기 조각 공개, 챕터 완성 시 반전

---

### 11. 경제 및 수익화 구조

**자원 종류**:
- 에너지: 플레이 비용, 분당 1개 자연 재생(최대 30개)
- 코인: 소프트 화폐, 특정 오브젝트 즉시 구매 가능
- 젬(Gem): 하드 화폐, 에너지 즉시 충전/코인 구매/희귀 오브젝트

**자원 획득 경로**:
- 에너지: 시간 재생 / 광고 시청(5~10개) / 젬 소비 / 이벤트 보상
- 코인: 미션 보상, 일일 보너스
- 젬: 유료 구매, 소량 이벤트 보상

**수익화 접점**:
- 보상형 광고: 에너지 보충 (가장 빈번한 접점) → Loss Avoidance 동인
- 인앱 결제: 젬 팩, 에너지 번들, 프리미엄 이벤트 통행증
- **핵심**: 에너지 소진 시점 = 서사가 막힌 시점 → "다음 이야기를 보고 싶다"는 욕구와 결합

**에너지 시스템**:
- 최대 30개, 분당 1개 재생 → 한 세션 약 10~20분 집중 플레이 가능
- 세션 간 간격 유도: 에너지 소진 → 30분~2시간 후 재접속 → 하루 3~4회 접속 패턴

---

### 12. 디자인 기둥

1. **서사가 병합에 목적을 부여**: 단순 병합이 아니라 "이걸 만들어야 저택이 복원된다"
2. **발견의 연속**: 병합 트리 탐색 = 새 오브젝트 발견 = 수집 욕구 충족
3. **공간의 시각 변화**: 복원 전후 대비가 소유감과 성취감을 만듦
4. **에너지로 세션 자연 조절**: 과몰입 방지 + 매일 재방문 유도
5. **서사 드리블링**: 이야기를 조금씩만 공개 → 다음 챕터 기대감 유지

---

### 13. 재사용 가능한 디자인 패턴

**패턴 1: 서사 드리블링 (Narrative Drip-feeding)**
- 설명: 서사를 한 번에 공개하지 않고 미션 완료마다 조금씩 공개
- 효과: "다음 이야기가 보고 싶다" 동기가 핵심 루프(병합) 반복 이유가 됨
- 예시: 공간 복원마다 에블린 독백 1~2줄, 챕터 완성마다 반전 영상

**패턴 2: 에너지 세션 조절 (Energy-Based Session Gating)**
- 설명: 에너지 제한으로 세션 길이를 자동 조절, 시간 재생으로 재방문 유도
- 효과: 과몰입 방지 + 하루 여러 번 짧게 재접속하는 습관 형성
- 예시: 에너지 30개 = 약 10~15분 집중 플레이, 이후 30분~2시간 대기

**패턴 3: 인벤토리 관리를 전략 깊이로 활용**
- 설명: 제한된 보드 공간에서 어떤 오브젝트를 보유할지 선택이 전략이 됨
- 효과: 단순 매칭을 넘어 "역산 계획"이라는 인지적 참여 추가
- 예시: 미션 목표 오브젝트를 보고 병합 트리를 거꾸로 추적하여 소환 결정

---

### 14. 프로토타입 범위 추천

**MVP**:
- 5×7 보드, 드래그 병합, 오브젝트 레벨 1~5 (병합 트리 고정)
- 에너지 소환 시스템 (에너지 소비 → 새 오브젝트 등장)
- 단순 미션 3개 (오브젝트 X개 제출)
- 미션 완료 시 단순 이미지 변화 (Before/After)
- **검증 가설**: "병합으로 새 오브젝트를 발견하는 것 자체가 재미있는가?"

**알파**:
- 오브젝트 레벨 1~10, 카테고리 3종
- 저택 공간 3개 복원
- 서사 대화 10줄 (텍스트만)
- 에너지 시스템 완성

**베타**:
- 오브젝트 카테고리 6종+
- 저택 공간 15개 이상
- 서사 챕터 3개 (컷씬 포함)
- 수익화: 보상형 광고 + 젬 구매
- 이벤트 시스템 기초

## 장르 횡단 패턴 비교

세 게임에서 발견되는 공통 디자인 패턴 및 장르별 차이점.

### 공통 패턴

| 패턴 | Hades | Royal Match | Merge Mansion |
|------|-------|-------------|---------------|
| 실패 완충 장치 | 귀환 서사 | Near Miss 설계 | 에너지 재생 대기 |
| 장기 리텐션 동인 | Narrative (NPC 관계) | Ownership (왕국) | Narrative (미스터리 서사) |
| 메타 진행 | 거울 영구 업그레이드 | 왕국 건설 | 저택 복원 |
| 변수 보상 | 랜덤 축복 조합 | 랜덤 보드 배치 | 병합 결과 기대감 |
| 코어-메타 연결 | 런 자원 → 메타 소비 | 별 → 건설 직결 | 오브젝트 → 미션 → 서사 |

### 장르별 에스테틱 차이

- **로그라이크 액션(Hades)**: Challenge + Narrative가 핵심, 단기 세션에 많은 의사결정 밀도
- **하이브리드 캐주얼 퍼즐(Royal Match)**: Challenge + Ownership이 핵심, 코어-메타 연결의 직접성이 리텐션 열쇠
- **하이브리드 캐주얼 머지(Merge Mansion)**: Discovery + Narrative가 핵심, 서사 드리블링이 에너지 시스템과 결합하여 재방문 유도

### 수익화 방식 비교

| 항목 | Hades | Royal Match | Merge Mansion |
|------|-------|-------------|---------------|
| 수익화 모델 | 일회성 구매 | 광고 + IAP | 광고 + IAP |
| 핵심 동인 | 없음 (가챠/광고 없음) | Loss Avoidance (실패 후) | Scarcity (에너지 부족) |
| 코어 경험 영향 | 없음 | 실패 시 선택적 | 에너지 소진으로 세션 제한 |
