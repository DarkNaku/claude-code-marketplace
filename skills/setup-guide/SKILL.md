# setup-guide — Unity IDE 수동 작업 가이드

## 목적

코드 작성만으로 완료할 수 없고 **Unity Editor에서 사용자가 직접 조작해야 하는 작업**을
단계별 가이드로 문서화합니다.

가이드는 개발 흐름을 끊지 않도록 **작업 직전에 미리 작성**하고,
사용자가 가이드를 따라 IDE 설정을 마친 뒤 코드 작업을 재개할 수 있게 합니다.

---

## 가이드 작성이 필요한 상황

다음 중 하나라도 해당하면 이 스킬을 사용합니다.

| 상황 | 예시 |
|------|------|
| 패키지 설치·제거 | Package Manager에서 패키지 추가/삭제 |
| Assembly Definition 생성·참조 추가 | .asmdef 파일 생성, References 배열 수정 |
| Project Settings 변경 | Tags/Layers 추가, Physics 설정, Scripting Define Symbols |
| 씬·Hierarchy 구성 | 씬 생성, GameObject 배치, 컴포넌트 추가 |
| Prefab 설정 | Prefab 생성, Inspector 필드 연결 |
| ScriptableObject 에셋 생성 | Create Asset 메뉴로 SO 인스턴스 생성 |
| Addressables 설정 | 그룹 생성, 에셋 주소 지정, 빌드 |
| Input Action Asset 연결 | Input Actions 에셋 생성, Player Input 컴포넌트 연결 |
| Test Runner 설정 | Test Runner 창 열기, 테스트 어셈블리 등록 |
| Renderer / URP 설정 | URP Asset 생성, Renderer Feature 추가 |

## 가이드 작성 워크플로우

### 1. 필요한 작업 식별

코드 작성 중 IDE 수동 작업이 필요한 시점을 감지합니다.

```
Glob: Assets/**/*.asmdef          ← 기존 Assembly Definition 파악
Glob: Packages/manifest.json      ← 설치된 패키지 파악
Glob: Assets/**/*.asset           ← 기존 ScriptableObject 에셋 파악
Glob: ProjectSettings/*.asset     ← 현재 Project Settings 파악
```

프로젝트 현황을 파악한 뒤 **이미 완료된 설정은 가이드에서 제외**합니다.

### 2. 가이드 파일 작성

아래 출력 구조에 따라 `[slug].setup.md`를 작성합니다.

### 3. 사용자에게 알림

가이드 파일 저장 후 사용자에게 다음을 전달합니다:
```
[slug].setup.md 가이드를 작성했습니다.
가이드의 작업을 완료한 뒤 알려주시면 코드 작업을 이어서 진행합니다.
```

## 교정 모드 — [slug].setup-fix-[N].md 작성

setup-validator가 ❌/⚠️ 항목을 발견하면 **교정 모드**로 이 스킬을 호출합니다.
교정 가이드는 초기 가이드와 구조가 같지만 내용이 다릅니다.

### 교정 가이드 작성 원칙

- **실패 항목만 포함**: ✅ 통과 항목은 제외하고 ❌/⚠️/🔍 항목만 작성
- **현재 상태 명시**: "무엇이 잘못됐는지"를 먼저 설명
- **수정 경로 제시**: "어떻게 고쳐야 하는지" 정확한 단계 제공

### 교정 가이드 출력 구조

```markdown
# [기능명] Setup 교정 가이드 ([N]회차)

> 이전 검증(`[slug].setup-result-[N-1].md`)에서 발견된 불일치 항목을 수정합니다.
> 아래 항목만 수정한 뒤 완료를 알려주세요.
```

## 작업 유형별 작성 기준

상세 패턴은 [IDE 작업 패턴 레퍼런스](references/ide-tasks.md)를 참고합니다.

### 공통 규칙

- **메뉴 경로**는 `메뉴 > 하위메뉴 > 항목` 형식으로 표기
- **Inspector 필드**는 `컴포넌트명 > 필드명` 형식으로 표기
- **단축키**는 `Ctrl+S` / `Cmd+S` (Windows/Mac) 병기
- **확인 항목**은 완료 후 눈으로 검증 가능한 상태로 서술
- 선택 항목이 여러 개일 때는 **정확한 값**을 명시 (예: `Type: Default`)
- 이미 설정된 항목은 "이미 적용된 경우 생략" 문구 추가

---

## 모범 사례

1. **선작업 후코드**: 가이드를 먼저 완성한 뒤 코드 작업을 멈추고 사용자에게 전달
2. **현황 파악 후 작성**: Glob/Grep으로 이미 완료된 설정을 확인하고 불필요한 단계 제거
3. **정확한 경로 명시**: "Inspector에서 설정" 대신 `Window > General > Inspector` 같이 구체적으로 기술
4. **완료 기준 명확화**: 각 작업 끝에 "확인" 항목으로 성공 여부를 판단할 수 있게 작성
5. **버전 고려**: 패키지 버전에 따라 UI가 다를 수 있는 경우 버전 조건 명시
