# setup-validator — IDE 설정 완료 검증

## 목적

사용자가 `[slug].setup.md` 가이드의 작업을 완료했다고 알리면,
**프로젝트 파일을 직접 검사**하여 각 항목이 올바르게 수행되었는지 검증합니다.

❌/⚠️ 항목이 있으면 **setup-guide 스킬로 교정 가이드를 작성**하여 사용자에게 전달하고,
완료 보고 후 다시 검증하는 **반복 프로세스**를 모든 항목이 통과할 때까지 유지합니다.

- ✅ 검증됨: 파일 검사로 완료 확인
- ⚠️ 부분 완료: 일부만 확인되거나 값이 예상과 다름
- ❌ 미완료: 파일 검사 결과 작업이 수행되지 않음
- 🔍 수동 확인 필요: 파일 검사만으로 판정 불가 (Inspector 연결 등)

---

## 검증 워크플로우

### 1. 회차(N) 결정

```
Glob: [slug].setup-result-*.md
```

- 없으면 N = 1 (최초 검증)
- 있으면 가장 큰 번호 + 1

### 2. 검증 대상 가이드 파일 결정

- N = 1: `[slug].setup.md` 읽기
- N ≥ 2: 직전 교정 가이드 `[slug].setup-fix-[N-1].md` 읽기 (교정 항목만 검증)
  - 원본 `[slug].setup.md`도 함께 읽어 전체 맥락 유지

파일이 없으면 사용자에게 경로를 확인하도록 안내하고 중단합니다.

### 3. 작업 항목 파싱

가이드의 `## 작업 목록` 섹션에서 체크리스트 항목을 추출합니다.
각 `## 작업 N.` 섹션을 읽어 **검증에 필요한 기대값**을 파악합니다:
- 예상 파일 경로
- 예상 패키지명·버전
- 예상 필드값·심볼명·레이어명

### 4. 항목별 파일 검사

항목 유형에 따라 아래 [검증 방법 레퍼런스](references/validation-methods.md)의 쿼리를 적용합니다.

### 5. 결과 리포트 저장

`[slug].setup-result-[N].md`로 저장합니다.

### 6. 후속 처리 — 완료율 기반 분기

아래 **완료율별 후속 처리** 섹션에 따라 진행합니다.

## 완료율별 후속 처리

### 100% — 모두 통과

1. 사용자에게 완료 알림
2. 코드 작업 즉시 재개

---

### 100% 미만 — 교정 가이드 작성 후 재검증 반복

❌/⚠️ 항목이 하나라도 있으면 다음 절차를 수행합니다.

**1. 불일치 정보 수집**

각 ❌/⚠️ 항목에 대해 다음을 정리합니다:
- 작업 제목
- 발견 내용 (현재 상태)
- 기대 내용 (가이드 요구사항)
- 차이점 요약

**2. setup-guide 스킬로 교정 가이드 작성**

setup-guide 스킬을 **교정 모드**로 호출하여 `[slug].setup-fix-[N].md`를 작성합니다.

교정 가이드에는 다음이 포함되어야 합니다:
- ❌/⚠️ 항목만 포함 (✅ 항목은 제외)
- 각 항목마다 **"현재 상태"** → **"올바른 상태"** 대비
- 차이를 수정하기 위한 구체적인 단계
- 🔍 항목은 수동 확인 방법 명시

**3. 사용자에게 교정 가이드 전달**

```
[slug].setup-fix-[N].md 교정 가이드를 작성했습니다.

[N]회차 검증 결과: 완료율 NN% ([미완료 수]개 항목 수정 필요)
교정 가이드의 작업을 완료한 뒤 알려주시면 다시 검증합니다.
```

**4. 사용자 완료 보고 후 setup-validator 재실행**

사용자가 완료를 알리면 N+1회차로 setup-validator를 다시 실행합니다.
이 반복은 **완료율 100%가 될 때까지** 계속됩니다.

---

### 🔍 수동 확인 필요 항목 처리

- 100% 외 항목이 모두 🔍인 경우: 사용자에게 수동 확인 목록을 전달하고 확인 완료 보고를 요청
- 수동 확인 완료 보고를 받으면 해당 항목을 ✅로 처리하고 최종 완료 판정

## 모범 사례

1. **가이드 기대값 우선**: 검증 기준은 setup.md에 명시된 값 — 임의로 기준을 완화하지 않음
2. **파일 검사 범위 명시**: 어떤 파일의 어떤 항목을 확인했는지 결과에 기록
3. **재시도 방법 구체화**: ❌ 항목은 "다시 해보세요" 대신 정확한 메뉴 경로와 값을 제시
4. **수동 확인 항목 분리**: 파일만으로 판정 불가능한 항목은 🔍로 명확히 구분
5. **재검증 유도**: 미완료 항목 수정 후 setup-validator를 다시 실행하도록 안내
