# MVP 설계 패턴

장르별 MVP 범위 정의 가이드와 자주 하는 실수 정리.

---

## 장르별 MVP 최소 기능 목록

### 로그라이크 서바이벌 (Vampire Survivors류)

**핵심 재미**: 자동 공격으로 적을 물리치며 레벨업 시 스킬을 조합하는 재미

**MVP 필수 기능**:
```
플레이어:
□ 8방향 이동
□ 자동 공격 무기 1종
□ 경험치 수집 + 레벨업

적:
□ 플레이어 추적 AI 1종
□ 스폰 시스템 (시간 기반)

게임 루프:
□ 레벨업 시 스킬 선택 (3개 중 1개)
□ HP 시스템 + 사망
□ 타이머 (생존 목표)
□ 재시작
```

**의도적으로 제외**:
- 무기 다양성 (MVP는 1종으로 충분)
- 메타 진행 (영구 업그레이드)
- 다양한 적 종류
- 세이브/로드
- 배경음악, 완성된 이펙트

**MVP 성공 기준**: 플레이어가 레벨업 선택 화면에서 "어떤 걸 골라야 하지?" 고민하면 성공

---

### 덱빌딩 카드게임 (Slay the Spire류)

**핵심 재미**: 카드 조합으로 시너지를 만들어 강력한 한 방을 터뜨리는 재미

**MVP 필수 기능**:
```
전투:
□ 에너지 시스템 (턴당 3)
□ 카드 드로우/플레이/버리기
□ 공격 카드 (데미지)
□ 방어 카드 (블럭)
□ 적 인텐트 표시 (다음 행동 예고)

덱:
□ 시작 덱 10장
□ 전투 보상으로 카드 추가 (3개 중 1개)

진행:
□ 전투 3~5회 연속
□ 체력 유지 (전투 사이에 회복 없음)
□ 게임 오버 + 재시작
```

**의도적으로 제외**:
- 캐릭터 여러 명
- 유물 시스템
- 맵/경로 선택
- 카드 업그레이드
- 상점

**MVP 성공 기준**: 플레이어가 "이 카드가 저 카드랑 잘 맞네" 시너지를 스스로 발견하면 성공

---

### 타워 디펜스

**핵심 재미**: 제한된 자원으로 타워를 배치하여 적의 경로를 막는 전략적 재미

**MVP 필수 기능**:
```
타워:
□ 타워 배치 시스템
□ 타워 1~2종 (사거리, 공격속도, 데미지 차이)
□ 타워 업그레이드 1단계

적:
□ 경로 이동 AI (웨이포인트)
□ 적 1~2종 (속도, HP 차이)
□ 웨이브 시스템 (3~5 웨이브)

경제:
□ 골드 (적 처치 획득, 타워 구매 소비)
□ 기지 HP (0이 되면 게임 오버)

게임 루프:
□ 웨이브 시작/종료
□ 배치 페이즈 / 전투 페이즈
□ 게임 오버 + 재시작
```

**의도적으로 제외**:
- 맵 다양성
- 특수 능력 타워
- 보스 웨이브
- 스킬/스펠

**MVP 성공 기준**: 플레이어가 "여기에 타워를 놓으면 될 것 같은데" 배치 전략을 스스로 세우면 성공

---

### 플랫포머 액션

**핵심 재미**: 정확한 점프와 전투로 장애물을 극복하는 반응속도 기반 재미

**MVP 필수 기능**:
```
플레이어:
□ 좌우 이동 + 점프
□ 기본 공격 1종
□ 적 피격 + HP 시스템

레벨:
□ 플랫폼 5~10개로 구성된 스테이지 1개
□ 골인 지점 (스테이지 클리어)
□ 낙사 처리

적:
□ 단순 패턴 적 1종 (좌우 이동)
□ 처치 피드백 (이펙트)

게임 루프:
□ 스테이지 클리어
□ 사망 + 재시작 (스테이지 처음부터)
```

**의도적으로 제외**:
- 다양한 이동 기술 (대시, 더블 점프)
- 여러 스테이지
- 아이템 시스템
- 보스

**MVP 성공 기준**: 플레이어가 스테이지를 클리어한 직후 "다시 해볼게요"라고 하면 성공

---

### 소셜 디덕션 (Among Us류)

**핵심 재미**: 누가 임포스터인지 추리하고 설득하는 사회적 긴장감

**MVP 필수 기능**:
```
역할:
□ 역할 배정 (크루원/임포스터)
□ 킬 기능 (임포스터)

임무:
□ 간단한 임무 3~5종
□ 임무 완료 체크

회의:
□ 시체 발견 신고
□ 투표 시스템
□ 플레이어 추방

승리 조건:
□ 크루원 승리 (임무 완료 OR 임포스터 추방)
□ 임포스터 승리 (크루원 수 동등)
```

**의도적으로 제외**:
- 음성 채팅 (텍스트 채팅으로 대체)
- 다양한 사보타지
- 다양한 맵
- 커스터마이징

**MVP 성공 기준**: 임포스터가 아닌 플레이어가 잘못 추방됐을 때 "억울해!"라는 반응이 나오면 성공

---

### 퍼즐 게임

**핵심 재미**: 규칙을 파악하고 적용하여 문제를 해결하는 성취감

**MVP 필수 기능**:
```
핵심 메카닉:
□ 핵심 규칙 1가지 완전 구현
□ 시각적으로 명확한 피드백

레벨:
□ 난이도 순서로 레벨 5~10개
□ 첫 레벨은 규칙 학습용 (튜토리얼 텍스트 없이)
□ 마지막 레벨은 규칙의 조합

게임 루프:
□ 레벨 클리어 인식
□ 다음 레벨 진행
□ 리셋 기능
```

**의도적으로 제외**:
- 힌트 시스템
- 스타 평가
- 다양한 메카닉 조합

**MVP 성공 기준**: 설명 없이 첫 레벨에서 규칙을 스스로 파악하면 성공

## 자주 하는 MVP 범위 과대 추정 실수

### 실수 1: "콘텐츠가 너무 부족하면 재미없을 것 같아"

**현실**: 핵심 재미가 없으면 콘텐츠가 100개여도 재미없음. 핵심 재미가 있으면 1개로도 검증 가능.

**해결책**: "이 기능이 없으면 핵심 재미를 경험할 수 없는가?"를 질문. 그렇지 않으면 제외.

---

### 실수 2: "UI가 너무 투박하면 플레이어가 이해 못할 것 같아"

**현실**: 임시 UI로도 충분히 플레이 테스트 가능. UI 완성도는 베타에서.

**해결책**: 텍스트 레이블, 단색 버튼으로 MVP 구성. 플레이어는 생각보다 잘 적응함.

---

### 실수 3: "세이브가 없으면 사람들이 안 할 것 같아"

**현실**: MVP 플레이 테스트는 15~30분. 세이브 불필요.

**해결책**: MVP에서는 세이브 없이 단일 세션으로 테스트. 알파에서 추가.

---

### 실수 4: "메타 진행이 없으면 리플레이성이 없어"

**현실**: 핵심 루프의 재미가 있으면 메타 없이도 여러 번 함. 메타는 장기 리텐션을 위한 것.

**해결책**: 핵심 루프 1회 경험으로 재미 검증 후 메타 진행 추가.

---

### 실수 5: "그래픽이 좋지 않으면 사람들이 안 봐줄 것 같아"

**현실**: 플레이 테스트에서 그래픽은 재미 검증에 영향을 거의 주지 않음.

**해결책**: Unity 기본 도형(Cube, Sphere)으로 MVP. 아트는 핵심 재미 검증 후.

---

## 핵심 재미 검증 방법

### 플레이 테스트 설계

**테스터 선정**:
- MVP: 팀원, 친구/가족 (편향 있어도 됨)
- 알파: 타겟 유저와 유사한 사람 (게이머)
- 베타: 실제 타겟 유저

**관찰 방법**:
```
1. 테스터에게 설명하지 말 것 (처음 5분)
2. 테스터의 행동과 표정 관찰
3. 소리 내어 생각하기 요청 ("지금 뭐 하려고 하세요?")
4. 막히는 지점 기록
5. 재시작 버튼을 스스로 누르는지 관찰
```

**핵심 질문**:
- "가장 재밌었던 순간은 언제였나요?"
- "다시 하고 싶나요? 이유는?"
- "뭔가 답답하거나 이해 안 된 부분이 있었나요?"

### 핵심 재미 검증 지표

| 지표 | MVP 기준 | 비고 |
|---|---|---|
| 자발적 재시작률 | 테스터의 50% 이상 | 강제 재시작 제외 |
| 핵심 루프 이해 시간 | 3분 이내 | 설명 없이 |
| 세션당 플레이 시간 | 계획 세션 시간의 150% | 더 오래 하면 재미있음 |
| "다시 하고 싶다" 응답률 | 60% 이상 | |

## MVP 일정 추정 가이드

### 팀 규모별 MVP 기간

| 팀 규모 | MVP 예상 기간 | 비고 |
|---|---|---|
| 솔로 개발자 | 2~4주 | 주말 작업 기준 2배 |
| 2인 (프로그래머 + 디자이너) | 1~2주 | |
| 3~5인 | 1주 | 역할 분담 명확히 |

|---|---|---|
| 캐릭터 이동 (2D) | 2~4시간 | |

| HP 시스템 | 2~4시간 | |
| 레벨업 시스템 | 4~8시간 | |
