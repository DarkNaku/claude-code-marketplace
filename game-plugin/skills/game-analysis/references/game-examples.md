# 게임 분석 사례

실제 인기 게임들의 상세 분석 사례.

## Hades - 로그라이크 액션 RPG

### 게임 개요

```
게임: Hades
장르: 로그라이크 액션 RPG
플랫폼: PC, Console
개발사: Supergiant Games
타겟 유저: 하드코어 액션 게임 유저

핵심 경험:
신화 속 지하세계를 탈출하려는 불사신의 반복적 도전,
매 시도마다 성장하며 스토리가 진행되는 내러티브 로그라이크
```

### 핵심 게임플레이 루프

**마이크로 루프** (초 단위):
```
적 조우 → 공격/회피 → 적 처치 → 다음 방
```

**메조 루프** (분 단위):
```
방 선택 → 전투 → 보상 획득 → 빌드 강화 → 다음 방
```

**매크로 루프** (세션, 30~60분):
```
탈출 시도 → 사망 → 하데스 귀환 → NPC 대화 → 영구 업그레이드 → 재도전
```

**메타 루프** (장기):
```
스토리 진행 → 캐릭터 관계 발전 → 새 무기 언락 → 엔딩 도달
```

### 주요 시스템 분해

#### 전투 시스템
- **무기 6종**: 검, 창, 방패, 활, 권총, 주먹
- **공격 타입**: 기본 공격 (연타), 특수 공격 (차지), 대시 공격
- **캐스트**: 원거리 투사체
- **콜**: 강력한 광역 스킬
- **타이밍 기반**: 패턴 암기 + 반응 속도

#### 빌드 시스템 (신의 축복)
- **12명의 올림포스 신**: 각각 고유한 능력 제공
- **해머 업그레이드**: 무기 메커닉 변경
- **듀오 축복**: 특정 신 조합으로 강력한 시너지
- **선택의 제약**: 방마다 제한된 선택지
- **빌드 다양성**: 수천 가지 조합

#### 진행 시스템
- **던전 구조**: 4개 지역 (타르타로스 → 아스포델 → 엘리시움 → 스틱스)
- **각 지역 특징**: 고유한 적, 보스, 환경
- **방 선택**: 보상 미리보기 (돈, 체력, 축복 등)
- **보스 4명**: 각기 다른 패턴

#### 메타 진행
- **다크니스**: NPC 호감도 상승
- **암브로시아**: 키프샤케 언락 (영구 능력)
- **타이탄의 피**: 거울 업그레이드 (영구 능력치)
- **다이아몬드**: 하우스 장식 + 특수 언락

### 의사결정 구조

**방 선택 의사결정**:
- 현재 빌드에 필요한 축복 vs 체력 회복
- 안전한 경로 vs 보상 좋은 위험 경로
- 시간제한 방 (보상 2배, 난이도 상승)

**전투 중 의사결정**:
- 공격 vs 회피 타이밍
- 특수 공격 사용 시점 (자원 소모)
- 콜 사용 타이밍 (1회 한정)

**빌드 구성 의사결정**:
- 신 축복 선택 (랜덤 3개 중 1개)
- 듀오 조합 노리기 vs 안전한 선택
- 해머 선택 (무기 특성 변경)

### 난이도 설계

**적응형 난이도**:
- 플레이어 진행에 따라 적 강화
- Heat 시스템: 자발적 난이도 상승 (보상 증가)

**학습 곡선**:
- 첫 탈출: 10~15시간 필요
- 패턴 암기: 각 보스당 5~10회 사망
- 마스터리: 최고 난이도 클리어 100시간+

**보상 곡선**:
- 초반: 빠른 메타 진행 (대화, 언락)
- 중반: 첫 탈출 성공 (20~30시간)
- 후반: 숨겨진 엔딩, 모든 관계 완성 (100시간+)

### UX 특징

**타격감**:
- 무기별 차별화된 타격 효과
- 크리티컬 시 시간 느려짐 효과
- 적 처치 시 폭발 이펙트

**정보 전달**:
- 적 패턴 명확한 텔레그래프 (바닥 표시)
- 데미지 숫자 크고 명확
- 위험 요소 빨간색 강조

**내러티브 통합**:
- 사망 = 귀환 (스토리적 정당화)
- 매 귀환마다 새로운 대화
- 진행에 따라 NPC 관계 변화

### 디자인 기둥

1. **진입장벽은 낮게, 숙련도는 높게**: 쉽게 배우고 어렵게 마스터
2. **실패도 진행**: 사망해도 스토리와 관계 진행
3. **다양성과 일관성**: 수천 가지 빌드, 하지만 모두 밸런스
4. **내러티브 중심**: 게임플레이와 스토리 완벽 통합
5. **높은 리플레이성**: 매번 다른 경험

### Unity 구현 모듈

```
GameManager
├── RunManager
│   ├── RoomGenerator (절차적 방 배치)
│   ├── RewardSystem (보상 선택)
│   └── ProgressTracker
├── CombatSystem
│   ├── PlayerController (ARPG 전투)
│   ├── WeaponManager (6종 무기)
│   ├── BoonSystem (신의 축복)
│   └── EnemyAI (패턴 기반)
├── MetaProgressionSystem
│   ├── MirrorUpgrade (영구 능력치)
│   ├── RelationshipSystem (NPC 호감도)
│   ├── WeaponUnlock
│   └── HeatSystem (난이도 조절)
├── NarrativeSystem
│   ├── DialogueManager
│   ├── CharacterRelationship
│   └── StoryProgression
└── UISystem
    ├── BoonSelectionUI
    ├── HUD (HP, 콜 게이지)
    └── CodexSystem (도감)
```

### 프로토타입 범위

**MVP** (2주):
- 플레이어 기본/특수 공격 + 대시
- 적 1종 + 간단한 패턴
- 방 3개 연속 진행
- 신의 축복 3종 (간단한 능력치 증가)
- 사망 시 재시작

**알파** (1개월):
- 무기 2종
- 적 5종 + 보스 1명
- 4개 지역 (간소화)
- 축복 20종 + 시너지 시스템
- 기본 메타 진행 (거울 업그레이드)

**베타** (3개월):
- 무기 6종 완성
- 적 30종 + 보스 4명
- 전체 지역 완성
- 축복 100종+ + 듀오/레전더리
- NPC 시스템 + 대화
- Heat 시스템

## Slay the Spire - 덱빌딩 로그라이크

### 게임 개요

```
게임: Slay the Spire
장르: 덱빌딩 로그라이크 카드 게임
플랫폼: PC, Mobile, Console
개발사: Mega Crit Games
타겟 유저: 전략 게임 유저, 카드 게임 팬

핵심 경험:
카드를 모으고 조합하여 강력한 덱을 구축,
최적화된 시너지로 점점 강해지는 적을 물리치는 전략적 재미
```

### 핵심 게임플레이 루프

**전투 루프** (분 단위):
```
손패 받기 → 카드 선택 → 플레이 → 에너지 소모 → 효과 발동 → 턴 종료
```

**방 루프** (분 단위):
```
방 선택 → 전투/이벤트 → 보상 획득 → 덱 강화 → 다음 방
```

**런 루프** (30~60분):
```
캐릭터 선택 → 1층 진행 → 보스 → 2층 → 보스 → 3층 → 최종 보스
```

### 주요 시스템 분해

#### 덱빌딩 시스템
- **시작 덱**: 캐릭터별 10장 (기본 공격 5, 방어 5)
- **카드 추가**: 전투 보상으로 선택
- **카드 제거**: 상점에서 골드로 구매
- **카드 업그레이드**: 캠프파이어에서 강화
- **덱 최적화**: 많을수록 좋지 않음 (희석 문제)

#### 전투 시스템
- **에너지**: 턴당 3 (기본)
- **카드 타입**: 공격, 스킬, 파워
- **상태 효과**: 약화, 취약, 힘, 민첩성 등
- **유물**: 영구 능력 (런 내)
- **포션**: 소모성 아이템

#### 캐릭터 3종
**아이언클래드** (전사):
- 컨셉: 고HP, 자해형 빌드
- 특징: 체력 회복, 방어력, 공격력 버프

**사일런트** (도적):
- 컨셉: 독/샤이브(0코스트 카드) 중심
- 특징: 카드 드로우, 약화, 연속 공격

**디펙트** (로봇):
- 컨셉: 오브(지속 효과) 중심
- 특징: 번개/서리 오브, 자동화

#### 진행 구조
- **3개 층**: 각 15~17개 방
- **방 타입**: 일반 전투, 엘리트, 상점, 보물, 캠프파이어, 이벤트
- **경로 선택**: 맵에서 경로 선택 (리스크/리워드)
- **보스 3명**: 각 층 마지막

### 의사결정 구조

**카드 선택**:
- 덱에 추가 vs 스킵
- 시너지 맞는 카드 vs 즉시 강한 카드
- 공격 vs 방어 밸런스

**경로 선택**:
- 안전한 일반 전투 vs 위험한 엘리트 (유물 보상)
- 캠프파이어 (회복 vs 업그레이드)
- 상점 vs 이벤트

**전투 중 선택**:
- 어떤 카드를 먼저 플레이할지
- 방어 우선 vs 공격 우선
- 포션 사용 타이밍

### 시너지 시스템

**아이언클래드 빌드 예시**:

**버프 덱**:
```
핵심 카드:
- Inflame (힘 +2)
- Limit Break (힘 2배)
- Heavy Blade (힘만큼 추가 데미지)

시너지:
- 힘 스택 → 공격력 기하급수적 증가
- 방어 카드 없어도 한 방에 끝냄
```

**블럭 덱**:
```
핵심 카드:
- Barricade (방어력 유지)
- Entrench (방어력 2배)
- Body Slam (방어력만큼 데미지)

시너지:
- 방어력 무한 축적 가능
- 방어 = 공격
```

### 수치 밸런싱

**데미지 스케일링**:
```
1층 적:
- 일반: HP 30~50, 데미지 5~10
- 엘리트: HP 100~150, 데미지 15~20
- 보스: HP 250, 다양한 패턴

3층 적:
- 일반: HP 100~200, 데미지 20~40
- 엘리트: HP 300~500, 데미지 40~60
- 보스: HP 500~800, 복잡한 패턴
```

**플레이어 성장**:
```
시작: HP 70, 기본 공격 6 데미지
1층 후: HP 80, 평균 데미지 10~15
2층 후: HP 90, 평균 데미지 20~30
3층 후: HP 100, 평균 데미지 40~50+ (시너지)
```

### 디자인 기둥

1. **선택과 결과**: 모든 선택이 덱에 영향
2. **시너지 탐구**: 카드 조합 발견의 재미
3. **적응형 전략**: 주어진 카드로 최선의 덱 구성
4. **리스크 관리**: 탐욕 vs 안전
5. **정보 기반 결정**: 모든 정보 공개 (운보다 전략)

### Unity 구현 모듈

```
GameManager
├── RunManager
│   ├── MapGenerator (절차적 맵)
│   ├── PathingSystem
│   └── EncounterManager
├── DeckSystem
│   ├── DeckManager (카드 리스트)
│   ├── DrawPile / DiscardPile
│   ├── HandManager
│   └── ExhaustPile
├── CombatSystem
│   ├── TurnManager (플레이어/적 턴)
│   ├── CardPlaySystem (카드 효과 처리)
│   ├── EnergySystem
│   └── StatusEffectManager
├── RewardSystem
│   ├── CardReward
│   ├── RelicReward
│   └── GoldReward
├── EnemyAI
│   ├── IntentSystem (적 행동 예고)
│   ├── PatternManager
│   └── EnemyAbilities

    ├── HandUI (카드 표시)
    ├── EnergyDisplay
    ├── IntentDisplay
    └── MapUI
```

### 프로토타입 범위

**MVP** (2주):
- 덱 10장 (공격 5, 방어 5)
- 에너지 시스템
- 카드 드로우/플레이/버리기
- 적 1종 (간단한 패턴)
- 전투 5회 연속

**알파** (1개월):
- 카드 30장 (다양한 효과)
- 적 10종 + 인텐트 시스템
- 1층 완성 (15개 방 + 보스)
- 유물 10종
- 카드 선택 보상 시스템

**베타** (3개월):
- 캐릭터 3종
- 카드 200장+
- 3개 층 완성
- 유물 100종+
- 포션 시스템
- 모든 이벤트 완성

## Among Us - 소셜 디덕션

### 게임 개요

```
게임: Among Us
장르: 소셜 디덕션 멀티플레이어
플랫폼: PC, Mobile
개발사: Innersloth
타겟 유저: 캐주얼 멀티플레이어 게이머

핵심 경험:
크루원으로 임무 수행하거나, 임포스터로 크루원을 제거하며,
투표와 토론으로 임포스터를 찾아내는 사회적 추론 게임
```

### 핵심 게임플레이 루프

**게임 루프** (5~15분):
```
역할 배정 → 임무 수행/살인 → 시체 발견/비상회의 → 토론 → 투표 → 반복
```

**소셜 루프**:
```
의심 제기 → 변명 → 증거 제시 → 설득 → 투표
```

### 역할 시스템

**크루원** (70~90%):
- 목표: 모든 임무 완료 OR 임포스터 추방
- 능력: 임무 수행, 비상회의 소집
- 사망 시: 유령으로 계속 임무 수행 가능

**임포스터** (10~30%):
- 목표: 크루원 수 = 임포스터 수
- 능력: 살인, 사보타지, 벤트 이동
- 가짜 임무: 크루원 행세

### 주요 메커닉

#### 임무 시스템
- **임무 타입**:
  - 단순 (1초): 스위치 켜기
  - 짧은 (3~5초): 카드 긁기
  - 긴 (10초+): 다운로드, 연료 충전
  - 여러 단계: 샘플 채취 (1분 대기)

- **임무 위치**: 맵 전역 분산
- **진행도**: 화면 상단 바로 표시

#### 사보타지 시스템
- **타입**:
  - 조명 끄기 (시야 감소)
  - 산소 고갈 (30초 내 해결 필요)
  - 원자로 붕괴 (30초 내 해결 필요)
  - 통신 교란 (임무 위치 숨김)

- **효과**: 크루원 강제 이동, 혼란 유발

#### 회의 시스템
- **시작 조건**:
  - 시체 발견 (Report)
  - 비상 버튼 (게임당 1회/인)

- **토론 시간**: 설정 가능 (30~120초)
- **투표 시간**: 설정 가능 (15~60초)
- **결과**: 최다 득표자 추방 (동률 시 아무도 추방 안됨)

### 정보 비대칭

**크루원이 아는 것**:
- 자신의 임무 목록
- 시체 위치 (발견 시)
- 다른 플레이어 위치 (시야 내)

**임포스터가 아는 것**:
- 다른 임포스터가 누구인지
- 벤트 위치
- 사보타지 위치
- 킬 쿨다운

**모두가 모르는 것**:
- 타인의 역할
- 사보타지 진행 (임포스터 제외)

### 게임 밸런스

**10인 게임 기준**:
```
크루원: 8명
임포스터: 2명

크루원 승리 조건:
1. 모든 임무 완료
2. 모든 임포스터 추방

임포스터 승리 조건:
1. 크루원 = 임포스터 수
```

**밸런싱 포인트**:
- 임무 수: 크루원당 1~3개
- 킬 쿨다운: 15~45초
- 시야 거리: 크루원 0.5~1.5 / 임포스터 1.0~2.0
- 플레이어 속도: 0.5~3.0

### UX 디자인

**색상 시스템**:
- 플레이어를 색으로 구분 (Red, Blue, Green 등)
- 토론 시 "빨강이 수상함" 식으로 언급

**미니맵**:
- 현재 위치 표시
- 사보타지 위치 표시
- 임무 위치 표시

**시각적 피드백**:
- 임무 완료: 초록 체크
- 킬: 강렬한 애니메이션
- 사보타지: 빨간 경보음

### 소셜 메커닉

**추론 플레이**:
- 누가 어디 있었는지 (알리바이)
- 누가 임무 했는지 (시각적 임무)
- 누가 수상한 행동을 했는지

**심리전**:
- 거짓말 탐지
- 압박 투표
- 희생양 전략 (크루원끼리 의심)

**협력 플레이**:
- 함께 이동 (안전)
- 정보 공유
- 전략적 투표

### 디자인 기둥

1. **단순한 규칙**: 5분이면 학습 가능

## 공통 패턴 분석

위 3개 게임에서 발견되는 공통 디자인 패턴:

### 1. 짧은 게임 세션
- Hades: 30~60분
- Slay the Spire: 30~60분
- Among Us: 5~15분

**이유**: 현대 플레이어는 짧은 세션 선호, 리플레이 유도

### 2. 메타 진행
- Hades: 거울 업그레이드, NPC 관계
- Slay the Spire: 캐릭터 언락, Ascension
- Among Us: 없음 (순수 스킬 기반)

**이유**: 장기 동기 부여, 점진적 난이도 극복

### 3. 절차적 생성
- Hades: 방 배치, 보상 위치
- Slay the Spire: 맵, 카드 보상, 이벤트
- Among Us: 임무 위치, 역할 배정

**이유**: 리플레이성, 다양성

### 4. 리스크/리워드 선택
- Hades: 위험한 방 (높은 보상)
- Slay the Spire: 엘리트 vs 일반 전투
- Among Us: 킬 타이밍 (목격 위험)

**이유**: 플레이어 의사결정에 깊이 부여

### 5. 명확한 피드백
- Hades: 타격감, 이펙트
- Slay the Spire: 데미지 숫자, 인텐트 표시
- Among Us: 킬 애니메이션, 투표 결과

**이유**: 플레이어 행동의 즉각적 확인
