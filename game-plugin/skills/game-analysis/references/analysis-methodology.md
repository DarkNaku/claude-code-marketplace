# 게임 분석 방법론

게임을 체계적으로 분석하기 위한 상세 기법 가이드.

## 시스템 분해 기법

### 1. 플레이어 액션 추적

플레이어가 할 수 있는 모든 행동을 나열.

**단계**:
1. 입력 매핑 파악 (키보드, 마우스, 터치)
2. 각 입력에 대응하는 게임 내 액션 기록
3. 액션의 결과 (상태 변화) 기록

**예시: 플랫포머 게임**
```
입력        → 액션           → 결과
←/→ 키      → 이동           → 위치 변경
스페이스    → 점프           → 수직 속도 증가
Shift       → 대시           → 빠른 이동 + 무적
마우스 클릭 → 공격           → 데미지 + 적 넉백
E 키        → 상호작용        → 아이템 획득 / 대화
```

### 2. 상태 머신 추출

게임 오브젝트의 상태 전환을 분석.

**플레이어 상태 예시**:
```
Idle (대기)
├→ Run (이동 입력)
├→ Jump (점프 입력, 땅에 있을 때)
├→ Attack (공격 입력)
└→ Die (체력 0)

Run
├→ Idle (입력 없음)
├→ Jump (점프 입력)
└→ Slide (Shift + ↓)

Jump
├→ Idle (착지)
├→ DoubleJump (공중에서 점프)
└→ Fall (최고점 도달)
```

**추출 방법**:
1. 각 상태 식별
2. 상태 전환 조건 기록
3. 각 상태에서 가능한 액션 파악

### 3. 데이터 흐름 분석

자원과 정보가 어떻게 흐르는지 추적.

**예시: 로그라이크 게임**
```
적 처치
  ↓
경험치 젬 생성
  ↓
플레이어 접촉
  ↓
경험치 증가
  ↓
레벨업 (임계값 도달)
  ↓
스킬 선택
  ↓
능력치 영구 증가
```

**분석 포인트**:
- 자원 생성 조건
- 변환 비율 (젬 1개 = 경험치 5)
- 소비 방식
- 축적 여부

### 4. 의존성 그래프

시스템 간 의존 관계 파악.

**예시**:
```
WeaponSystem
├─ 의존 → PlayerStats (공격력)
├─ 의존 → EnemySystem (타겟)
└─ 의존 → ProjectilePool (투사체)

EnemySystem
├─ 의존 → PlayerPosition (추적)
├─ 의존 → HealthSystem (피해 계산)
└─ 생성 → LootSystem (드롭)
```

**활용**:
- 구현 순서 결정
- 모의 객체(Mock) 식별
- 순환 의존성 발견

## 수치 역설계 방법

### 1. 타이밍 측정

스톱워치나 영상 분석으로 시간 측정.

**측정 대상**:
- 공격 속도 (초당 타수)
- 이동 속도 (초당 이동거리)
- 쿨다운 시간
- 애니메이션 길이
- 게임 세션 길이

**예시: 공격 속도 계산**
```
10초 동안 공격 횟수: 15회
→ 공격 속도 = 1.5회/초
→ 공격 간격 = 0.67초

구현 시:
attackCooldown = 0.67f;
```

### 2. 수치 비율 추정

상대적 크기나 비율 파악.

**데미지 계산 역설계**:
```
관찰:
- 플레이어 HP: 100
- 기본 공격으로 적 처치: 5회
→ 적 HP 추정: 약 100 (플레이어와 동일?)

- 적 공격 1회로 HP 10 감소
→ 적 공격력: 10
→ 플레이어 방어 전 데미지: 10

- 방어구 착용 후 HP 7 감소
→ 방어력 30% (10 → 7, 30% 감소)
```

### 3. 진행 곡선 분석

난이도나 성장 속도 분석.

**레벨업 경험치 곡선**:
```
레벨 1→2: 100 EXP (기준)
레벨 2→3: 150 EXP (+50%)
레벨 3→4: 220 EXP (+47%)
레벨 4→5: 320 EXP (+45%)

패턴 파악:
- 지수 증가 (약 1.45배)
- 공식 추정: exp = 100 * (1.45 ^ (level - 1))
```

**적용**:
```csharp
public int GetRequiredExp(int level)
{
    return Mathf.RoundToInt(100 * Mathf.Pow(1.45f, level - 1));
}
```

### 4. 확률 추정

드롭률이나 크리티컬 확률 계산.

**방법**:
```
시행 횟수: 100회
아이템 드롭: 23회
→ 드롭률 약 23%

크리티컬 발생: 14회
→ 크리티컬 확률 약 14%
```

**더 정확한 추정**:
```
200회 시행 → 드롭 44회 (22%)
300회 시행 → 드롭 68회 (22.7%)
→ 실제 드롭률 추정: 20~25%
```

## UX 플로우 분석

### 1. 사용자 여정 매핑

플레이어 경험을 시간순으로 기록.

**예시: 첫 플레이 경험**
```
0:00 - 타이틀 화면
  ↓
0:10 - 게임 시작 클릭
  ↓
0:15 - 캐릭터 선택 화면
  │   └─ 초보자 가이드: "첫 번째 캐릭터 추천"
  ↓
0:30 - 튜토리얼 시작
  │   └─ 이동 연습 (강제)
  ↓
0:50 - 첫 적 조우
  │   └─ 자동 공격 설명
  ↓
1:30 - 첫 레벨업
  │   └─ 스킬 선택 UI 등장 (시간 정지)
  ↓
2:00 - 자유 플레이
```

**분석 포인트**:
- 첫 의미 있는 선택까지 시간 (1:30)
- 강제 튜토리얼 길이 (30초)
- 정보 제공 타이밍

### 2. 피드백 루프 분석

플레이어 액션 → 피드백 사이클.

**좋은 피드백 예시**:
```
플레이어 액션: 공격 버튼
  ↓ (즉시, 0.05초 이내)
시각 피드백: 캐릭터 공격 애니메이션
  ↓ (동시)
청각 피드백: 공격 사운드
  ↓ (0.1초 후)
적 반응: 피격 애니메이션 + 넉백
  ↓ (0.2초 후)
데미지 숫자 표시
  ↓ (1초 후)
적 처치 시 아이템 드롭
```

**측정 지표**:
- 피드백 지연 시간 (ms)
- 피드백 레이어 수 (시각, 청각, 햅틱)
- 피드백 강도 (파티클, 셰이크 정도)

### 3. 정보 계층 구조

UI 정보의 중요도와 배치 분석.

**예시: HUD 분석**
```
최우선 정보 (항상 보임):
- HP바 (화면 상단 왼쪽, 크고 명확)
- 미니맵 (우측 상단)

중요 정보 (필요 시 확인):
- 스킬 쿨다운 (화면 하단)
- 골드 (우측 상단)

부차적 정보:
- FPS (좌측 상단 작게)
- 키 가이드 (화면 하단 중앙)
```

**분석 방법**:
1. 스크린샷 촬영
2. 각 UI 요소의 위치, 크기 기록
3. 사용 빈도와 크기 상관관계 확인

## 밸런싱 분석

### 1. DPS 계산

초당 데미지 산출.

**예시**:
```
무기 A:
- 데미지: 10
- 공격속도: 2회/초
- DPS = 10 × 2 = 20

무기 B:
- 데미지: 25
- 공격속도: 0.7회/초
- DPS = 25 × 0.7 = 17.5

→ 무기 A가 더 효율적 (하지만 단발 데미지는 B가 강함)
```

**활용**:
- 무기 간 밸런스 비교
- 빌드 최적화
- 적 체력 설정 (DPS 대비)

### 2. 효율성 분석

자원 대비 효과 계산.

**업그레이드 효율**:
```
업그레이드 1단계:
- 비용: 100 골드
- 효과: 공격력 +5
- 효율: 0.05 공격력/골드

업그레이드 2단계:
- 비용: 300 골드
- 효과: 공격력 +7
- 효율: 0.023 공격력/골드

→ 1단계가 2배 이상 효율적
→ 후반 업그레이드는 비용 증가율 > 효과 증가율
```

### 3. 밸런스 곡선

게임 전체의 난이도 흐름 파악.

**생존 확률 곡선**:
```
시간대    생존율    분석
0-5분     100%     튜토리얼, 쉬움
5-10분    85%      첫 고비, 빌드 중요
10-15분   50%      중반 고비, 최적화 필요
15-20분   20%      후반, 고수만 생존
20분+     5%       극한 난이도
```

**적용**:
- 난이도 조절 포인트 식별
- 플레이어 이탈 시점 파악
- 보상 타이밍 조정

## 경쟁 분석 프레임워크

여러 게임을 비교 분석.

### 1. 기능 매트릭스

|기능|게임 A|게임 B|게임 C|
|---|------|------|------|
|자동 전투|O|X|O|
|메타 진행|O|O|X|
|멀티플레이|X|O|O|
|캐릭터 커스터마이징|높음|중간|낮음|
|콘텐츠 업데이트|월 1회|주 1회|분기 1회|

### 2. SWOT 분석

**게임 A (Vampire Survivors)**:
```
Strengths (강점):
- 낮은 진입장벽
- 높은 중독성
- 빠른 개발 사이클

Weaknesses (약점):
- 단순한 그래픽
- 제한된 콘텐츠
- PvP 없음

Opportunities (기회):
- 모바일 이식
- 협동 모드 추가
- IP 확장

Threats (위협):
- 많은 복제작
- 빠른 유행 소멸
- 낮은 진입장벽 = 경쟁 많음
```

### 3. 차별화 포인트 식별

**게임 A vs 게임 B**:
```
공통점:
- 로그라이크 진행
- 짧은 런 타임 (15~30분)
- 메타 업그레이드

차별점:
게임 A:
- 완전 자동 전투
- 빌드 중심
- 생존이 목표

게임 B:
- 수동 전투 (스킬 타이밍)
- 플레이어 스킬 중심
- 보스 처치가 목표

→ 타겟 유저: A는 캐주얼, B는 하드코어
```

### 분석 체크리스트

**게임 분석 완료 체크리스트**:
- [ ] 핵심 루프 3단계 정의 (마이크로/메조/매크로)
- [ ] 주요 시스템 5개 이상 분해
- [ ] 플레이어 액션 10개 이상 나열
- [ ] 수치 데이터 역설계 (공격력, HP, 속도 등)
- [ ] UX 플로우 매핑 (첫 5분)
- [ ] 보상 구조 분석 (즉각/단기/장기)
- [ ] 난이도 곡선 추정
- [ ] Unity 모듈 구조 설계
- [ ] 프로토타입 범위 정의
- [ ] 재사용 가능한 패턴 3개 이상 추출

**심화 분석 체크리스트**:
- [ ] 경쟁 게임 3개와 비교
- [ ] 플레이 영상 30분 이상 분석
- [ ] 커뮤니티 피드백 조사
- [ ] 수익 모델 분석 (F2P의 경우)
- [ ] 업데이트 히스토리 조사
- [ ] 기술적 제약 파악 (네트워크, 플랫폼 등)
