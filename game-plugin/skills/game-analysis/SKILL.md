# 게임 분석 스킬 - 레퍼런스 게임 분석

## 목적

레퍼런스 게임을 분석하여 **실행 가능한 디자인 인사이트**와 **구현 가능한 시스템**으로 변환합니다.

**추출 대상**:
- 핵심 게임플레이 루프
- 메커닉과 시스템 구조
- 플레이어 진행 경로
- UX 및 피드백 패턴
- 경제 구조 및 밸런싱
- 콘텐츠 파이프라인
- Unity 구현 모듈

## 분석 워크플로우

사용자가 레퍼런스 게임을 제공하면 다음 순서로 분석합니다:

1. **장르 및 디자인 목표 식별**: 게임이 해결하려는 문제와 타겟 경험
2. **핵심 루프 추출**: 플레이어가 반복하는 핵심 사이클
3. **시스템 분해**: 주요 게임플레이 시스템을 모듈별로 분리
4. **진행 구조 분석**: 플레이어 성장과 난이도 곡선
5. **의사결정 구조**: 플레이어가 내리는 의미 있는 선택
6. **UX/피드백 시스템**: 정보 전달 및 느낌 설계
7. **재사용 가능한 패턴**: 다른 게임에 적용 가능한 디자인 패턴
8. **구현 아키텍처**: Unity 시스템으로 변환

## 출력 구조

항상 다음 구조로 분석 결과를 제공합니다.

### 1. 게임 개요

```
게임: [게임 제목]
장르: [장르]
플랫폼: [플랫폼]
타겟 유저: [타겟 유저층]

게임 경험 요약: [한 문장으로 핵심 경험 설명]
```

**예시**:
```
게임: Vampire Survivors
장르: 로그라이크 서바이벌
플랫폼: PC, 모바일
타겟 유저: 캐주얼 액션 게임 유저

게임 경험 요약:
끊임없이 몰려오는 적을 자동 공격으로 물리치며 성장하는
카타르시스와 빌드 조합의 재미를 제공하는 캐주얼 로그라이크
```

### 2. 핵심 게임플레이 루프

가장 작은 반복 단위부터 큰 단위까지 계층적으로 설명.

**마이크로 루프** (초 단위):
```
이동 → 적 조우 → 자동 공격 → 경험치 획득 → 반복
```

**메조 루프** (분 단위):
```
레벨업 → 스킬 선택 → 강해진 빌드로 생존 → 다음 레벨업 → 반복
```

**매크로 루프** (게임 세션):
```
게임 시작 → 15분 생존 도전 → 사망 → 메타 업그레이드 → 새 빌드 시도
```

**분석 포인트**:
- 플레이어 동기: 왜 루프를 반복하는가?
- 보상 타이밍: 언제 만족감을 주는가?
- 루프 지속시간: 한 사이클이 얼마나 걸리는가?

### 3. 플레이어 의사결정 구조

플레이어가 내리는 **의미 있는 선택**을 식별.

**예시**:
- **빌드 방향 선택**: 공격속도형 vs 광역공격형
- **레벨업 선택**: 신규 무기 vs 기존 무기 강화
- **이동 전략**: 안전한 외곽 vs 위험한 중앙 (아이템 밀집)
- **타이밍 판단**: 보스 처치 vs 도망

**좋은 의사결정의 특징**:
- 명확한 트레이드오프
- 즉각적인 피드백
- 전략적 깊이

### 4. 게임플레이 시스템 분해

#### 이동 시스템
- 8방향 이동 (WASD/조이스틱)
- 일정 속도 유지
- 이동속도 업그레이드 존재
- 장애물 충돌 없음

#### 전투 시스템
- **자동 공격**: 플레이어 조작 불필요
- **다중 무기**: 동시에 여러 무기 장착
- **적 상호작용**: 단순 충돌 기반
- **데미지 모델**: 고정 데미지 + 크리티컬

#### 자원 시스템
- **경험치 젬**: 적 처치 시 드롭
- **골드**: 런 종료 후 획득
- **체력**: 고정값, 회복 아이템으로 관리

#### 진행 시스템
- **런 내 진행**: 레벨업 (1~100)
- **메타 진행**: 골드로 영구 업그레이드
- **캐릭터 언락**: 특정 조건 달성
- **스테이지 언락**: 특정 시간 생존

### 5. 보상 구조

**즉각 보상** (초 단위):
- 경험치 젬 획득 (시각/사운드 효과)
- 적 처치 만족감

**단기 보상** (분 단위):
- 레벨업 (스킬 선택의 재미)
- 새 무기 획득

**장기 보상** (세션 종료):
- 골드 획득
- 새 캐릭터/스테이지 언락
- 통계 기록

**보상 빈도 분석**:
- 경험치: 1~3초마다
- 레벨업: 초반 10초, 후반 30초
- 아이템 드롭: 30초~1분
- 메타 보상: 세션당 1회

### 6. 난이도 곡선

**난이도 증가 방식**:
- **적 수 증가**: 시간이 지날수록 스폰율 상승
- **적 체력 증가**: 후반 적은 높은 체력
- **새로운 적 패턴**: 5분, 10분마다 신규 적 등장
- **보스 등장**: 특정 시간(10분, 20분)에 강력한 보스

**플레이어 성장 속도**:
- 초반: 빠른 성장 (1~10레벨)
- 중반: 안정적 성장 (10~50레벨)
- 후반: 느린 성장, 최적화 필요 (50~100레벨)

**밸런싱 포인트**:
- 10분: 첫 번째 고비 (생존 난이도 급상승)
- 20분: 빌드 완성 여부 판가름
- 30분: 극한 생존 도전

### 7. UX / 피드백 디자인

**타격감**:
- 적 피격 시 넉백
- 히트 이펙트 (파티클)
- 사운드 피드백
- 카메라 셰이크 (보스 처치)

**정보 전달**:
- HP바: 화면 상단 고정
- 경험치바: HP 아래 표시
- 레벨업: 시간 정지 + 선택 UI
- 미니맵: 적 위치 표시

**튜토리얼 구조**:
- 첫 30초: 이동만
- 1분: 첫 레벨업 강제 (스킬 선택 설명)
- 2분: 아이템 획득 안내
- 이후: 자연스러운 학습

### 8. 콘텐츠 구조

**스테이지 구성**:
- 기본 스테이지: Library (평지)
- 언락 스테이지: Dairy Plant (장애물), Gallo Tower (좁은 공간)
- 각 스테이지별 고유 메커닉

**콘텐츠 생성 방식**:
- 절차적 적 배치
- 고정된 맵 레이아웃
- 타임라인 기반 이벤트 (보스 스폰)

### 9. 경제 시스템

**화폐**:
- **골드**: 런 종료 시 획득, 영구 업그레이드에 사용
- **획득량**: 생존 시간 + 적 처치 수

**소비처**:
- 캐릭터 언락: 1,000~10,000 골드
- 파워업: 100~5,000 골드
- 특수 아이템: 500~3,000 골드

**밸런싱**:
- 첫 런: 500 골드 (1개 업그레이드)
- 5회 런: 5,000 골드 누적 (핵심 업그레이드 완성)
- 20회 런: 대부분 언락

### 10. 디자인 기둥

게임을 지탱하는 3~5개의 핵심 원칙:

1. **접근성**: 원버튼 플레이, 낮은 진입장벽
2. **끊임없는 성장**: 매 초 강해지는 느낌
3. **빌드 다양성**: 수백 가지 스킬 조합
4. **높은 리플레이성**: 짧은 런 타임, 다양한 시도
5. **즉각적인 만족**: 화려한 이펙트, 빠른 피드백

### 11. 재사용 가능한 디자인 패턴

**패턴 1: 자동 전투 + 수동 빌드**
- 전투는 자동, 빌드 구성은 수동
- 캐주얼 + 전략의 조합

**패턴 2: 시간 기반 진행**
- 생존 시간이 목표
- 명확한 게임 길이 (15~30분)

**패턴 3: 메타 진행 루프**
- 런 실패해도 영구 업그레이드
- 점진적 난이도 극복

**패턴 4: 선택의 제한**
- 레벨업 시 3~4개 중 1개 선택
- 제한된 선택이 전략성 강화

**패턴 5: 시너지 시스템**
- 특정 스킬 조합으로 강력한 효과
- 빌드 탐구 동기 부여

### 13. 프로토타입 범위 추천

**MVP 프로토타입** (1주):
- 플레이어 8방향 이동
- 자동 공격 무기 1개 (예: 채찍)
- 단순 적 1종 (플레이어 추적)
- 경험치 시스템 + 레벨업
- 레벨업 시 스킬 선택 (3개 중 1개)
- 기본 UI (HP, EXP)
- 타이머 (15분)

**알파 프로토타입** (2~3주):
- 무기 5종 + 업그레이드
- 적 3종 + 보스
- 메타 진행 (골드 + 업그레이드)
- 스테이지 1개 완성
- 30분 생존 가능

**베타 프로토타입** (1~2개월):
- 무기 10종 + 진화 시스템
- 적 10종 + 보스 3종
- 스테이지 3개
- 캐릭터 5종
- 전체 메타 진행 완성

### 14. 추가 분석 (다중 레퍼런스)

여러 레퍼런스 게임 제공 시:

**비교 분석**:
- 공통 시스템 식별
- 차별화 포인트 분석
- 각 게임의 강점/약점

**하이브리드 디자인 제안**:
- 게임 A의 전투 + 게임 B의 진행
- 새로운 조합의 가능성

## 모범 사례

1. **실제 플레이 경험**: 직접 플레이하거나 플레이 영상 참고
2. **정량적 분석**: 수치를 최대한 역설계
3. **플레이어 관점**: 왜 재미있는지, 왜 중독성 있는지
4. **구현 가능성**: Unity에서 현실적으로 구현 가능한 범위
5. **우선순위**: 핵심 재미부터 식별
